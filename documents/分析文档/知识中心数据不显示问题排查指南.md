# 知识中心数据不显示问题排查指南

## 问题描述

前端知识中心页面看不到数据，即使数据库已经添加了数据。

---

## 一、已修复的问题

### 1.1 Repository 查询语法错误

**问题**：`MediaFileRepository` 中的 LIKE 查询语法错误

**修复前**：
```java
"(:query IS NULL OR m.title LIKE %:query%)"
```

**修复后**：
```java
"(:query IS NULL OR m.title LIKE CONCAT('%', :query, '%'))"
```

**说明**：JPQL 中不能直接使用 `%:query%`，需要使用 `CONCAT` 函数。

## 三、常见问题及解决方案

### 3.1 问题：后端返回空数组 `[]`

**可能原因**：
1. 数据库中没有数据
2. 查询条件不匹配
3. 数据格式不正确

**解决方案**：
1. **检查数据库**：确认 `media_files` 表中有数据
2. **检查数据格式**：确认 `type` 字段的值是枚举值之一
3. **检查分类**：确认 `category` 字段的值与前端分类匹配

**数据示例**：
```sql
INSERT INTO media_files (
    file_id, 
    title, 
    type, 
    category, 
    access_method,
    created_at, 
    updated_at
) VALUES (
    'KB001',
    '测试文档',
    'PDF',
    'sales',
    'SEAFILE',
    NOW(),
    NOW()
);
```

### 3.2 问题：CORS 跨域错误

**错误信息**：
```
Access to fetch at 'http://localhost:8080/api/cleaner-support/v2/media/files' 
from origin 'http://localhost:5173' has been blocked by CORS policy
```

**解决方案**：
1. **检查后端 CORS 配置**：确认 `CorsConfig` 已正确配置
2. **检查前端地址**：确认前端地址在允许列表中
3. **重启后端服务**：修改配置后需要重启

**CORS 配置检查**：
```java
// 确认 CorsConfig.java 存在且正确配置
@Configuration
public class CorsConfig {
    @Value("${app.cors.allowed-origins:http://localhost:5173,http://localhost:3000}")
    private String allowedOrigins;
    // ...
}
```

### 3.3 问题：数据格式不匹配

**错误现象**：
- 后端返回数据，但前端显示为空
- 浏览器控制台有警告信息

**检查点**：
1. **字段名匹配**：后端返回的字段名必须与前端期望的一致
   - 后端：`id`（对应 `fileId`）
   - 前端期望：`id`

2. **类型匹配**：
   - 后端：`type` 是字符串（如 "PDF"）
   - 前端期望：字符串

**验证方法**：
在浏览器控制台查看实际返回的数据：
```javascript
fetch('http://localhost:8080/api/cleaner-support/v2/media/files')
  .then(res => res.json())
  .then(data => {
    console.log('Response:', data);
    console.log('Data array:', data.data);
    if (data.data && data.data.length > 0) {
      console.log('First item:', data.data[0]);
      console.log('Fields:', Object.keys(data.data[0]));
    }
  });
```

### 3.4 问题：查询条件不匹配

**现象**：
- 选择"全部"分类时看不到数据
- 选择特定分类时看不到数据

**检查**：
1. **检查分类值**：确认数据库中的 `category` 值与前端分类映射一致
   - 前端分类：`sales`, `maintenance`, `product`, `company`, `training`
   - 数据库值：必须完全匹配（区分大小写）

2. **检查查询逻辑**：
   - "全部"分类：`category` 应该为 `null` 或 `undefined`
   - 特定分类：`category` 应该与数据库中的值完全匹配

**测试查询**：
```sql
-- 测试按分类查询
SELECT * FROM media_files WHERE category = 'sales';
SELECT * FROM media_files WHERE category = 'maintenance';
SELECT * FROM media_files WHERE category IS NULL;
```

### 3.5 问题：后端服务未启动或端口错误

**检查**：
1. **确认后端服务已启动**：访问 `http://localhost:8080/actuator/health`
2. **确认端口正确**：检查 `application.yml` 中的 `server.port`
3. **确认前端 API 地址正确**：检查 `.env` 或环境变量中的 `VITE_API_BASE_URL`

## 四、调试工具

### 4.1 后端调试

**添加日志**：

在 `MediaService.java` 中添加：
```java
@Service
public class MediaService {
    private static final Logger logger = LoggerFactory.getLogger(MediaService.class);
    
    public List<MediaFileSummaryResponse> searchFiles(String category, String query) {
        logger.info("searchFiles called: category={}, query={}", category, query);
        
        List<MediaFile> files;
        if (category != null || query != null) {
            files = mediaFileRepository.searchByCategoryAndQuery(category, query);
        } else {
            files = mediaFileRepository.findAll();
        }
        
        logger.info("Found {} files", files.size());
        files.forEach(f -> logger.debug("File: {}", f.getFileId()));
        
        return files.stream()
                .map(this::toSummaryResponse)
                .collect(Collectors.toList());
    }
}
```

### 4.2 前端调试

**在 KnowledgePage.tsx 中添加日志**：

```typescript
const loadFiles = useCallback(async () => {
  setLoading(true);

  try {

    console.log('Loading files:', { category, query }); // 添加日志

  }
```

## 五、数据验证 SQL

**用于验证数据库数据的 SQL 脚本**：

```sql
-- 1. 检查表结构
DESCRIBE media_files;

-- 2. 检查数据总数
SELECT COUNT(*) AS total FROM media_files;

-- 3. 检查必填字段
SELECT 
    COUNT(*) AS total,
    SUM(CASE WHEN file_id IS NULL THEN 1 ELSE 0 END) AS missing_file_id,
    SUM(CASE WHEN title IS NULL THEN 1 ELSE 0 END) AS missing_title,
    SUM(CASE WHEN type IS NULL THEN 1 ELSE 0 END) AS missing_type
FROM media_files;

-- 4. 检查分类分布
SELECT category, COUNT(*) AS count 
FROM media_files 
GROUP BY category;

-- 5. 检查类型分布
SELECT type, COUNT(*) AS count 
FROM media_files 
GROUP BY type;

-- 6. 查看前10条数据
SELECT 
    id,
    file_id,
    title,
    type,
    category,
    access_method,
    created_at
FROM media_files
ORDER BY created_at DESC
LIMIT 10;
```
