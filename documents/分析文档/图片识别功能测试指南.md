## 一、环境配置

### 1.1 后端配置

在 `application.yml` 或环境变量中配置以下参数：

```yaml
app:
  image-recognition:
    qwen-vl:
      api-url: https://dashscope.aliyuncs.com/api/v1/services/aigc/multimodal-generation/generation
      api-key: ${QWEN_VL_API_KEY}  # 必填：从阿里云DashScope获取
      model: qwen-vl-plus  # 可选：qwen-vl-max, qwen-vl-plus
      timeout: 30000
      max-retries: 3
    upload-dir: ./data/images  # 图片存储目录
    max-file-size: 10485760    # 10MB
    allowed-formats: jpg,jpeg,png,webp
```

**重要：** 需要从 [阿里云DashScope控制台](https://dashscope.console.aliyun.com/) 获取 API Key。

### 1.2 数据库初始化

确保数据库已创建 `image_recognitions` 表（JPA会自动创建，或手动执行SQL）：

```sql
CREATE TABLE IF NOT EXISTS image_recognitions (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    recognition_id VARCHAR(50) UNIQUE NOT NULL,
    user_id BIGINT NOT NULL,
    image_url VARCHAR(500) NOT NULL,
    image_path VARCHAR(500) NOT NULL,
    description TEXT,
    status VARCHAR(20) NOT NULL,
    error_message VARCHAR(500),
    created_at TIMESTAMP NOT NULL,
    updated_at TIMESTAMP NOT NULL,
    FOREIGN KEY (user_id) REFERENCES users(id),
    INDEX idx_user_status (user_id, status),
    INDEX idx_created_at (created_at DESC)
);
```

## 三、API 手动测试

### 3.1 使用 curl 测试

#### 步骤1：登录获取Token

```bash
curl -X POST http://localhost:8080/api/cleaner-support/v2/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "phone": "13800138000",
    "password": "your_password"
  }'
```

响应示例：
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {...}
  }
}
```

#### 步骤2：上传图片并识别

```bash
# 替换 YOUR_TOKEN 为实际token
curl -X POST http://localhost:8080/api/cleaner-support/v2/image-reco \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "image=@/path/to/your/image.jpg"
```

响应示例：
```json
{
  "code": 200,
  "message": "识别成功",
  "data": {
    "recognitionId": "IMG20240120001",
    "imageUrl": "/api/cleaner-support/v2/media/images/xxx.jpg",
    "description": "图片中显示一个清洁机器人，主刷部分被地毯缠绕...",
    "status": "completed",
    "createdAt": "2024-01-20T10:30:00"
  }
}
```

#### 步骤3：获取识别历史

```bash
curl -X GET "http://localhost:8080/api/cleaner-support/v2/image-reco/history?page=1&size=10" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

#### 步骤4：基于图片进行AI对话

```bash
curl -X POST http://localhost:8080/api/cleaner-support/v2/ai/chat/with-image \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "recognitionId": "IMG20240120001",
    "query": "这个问题怎么解决？",
    "conversationId": "conv_12345678"
  }'
```

**注意：** 这是SSE流式响应，需要使用支持流式处理的工具。

## 四、前端测试

### 4.1 在 React 组件中使用

```typescript
import { 
  uploadAndRecognizeImage, 
  sendAIMessageWithImage,
  getImageRecognitionHistory 
} from '@/services/api';

// 上传图片并识别
const handleImageUpload = async (file: File) => {
  try {
    const response = await uploadAndRecognizeImage(file);
    console.log('识别结果:', response);
    console.log('图片描述:', response.description);
    console.log('识别ID:', response.recognitionId);
  } catch (error) {
    console.error('识别失败:', error);
  }
};

// 基于图片进行AI对话
const handleChatWithImage = (recognitionId: string, query: string) => {
  const cancel = sendAIMessageWithImage(
    { recognitionId, query },
    (event) => {
      if (event.event === 'message' && event.answer) {
        console.log('AI回复:', event.answer);
      } else if (event.event === 'message_end') {
        console.log('对话结束');
      }
    },
    (error) => {
      console.error('对话失败:', error);
    },
    () => {
      console.log('对话完成');
    }
  );
  
  // 可以调用 cancel() 来取消请求
};

// 获取识别历史
const loadHistory = async () => {
  try {
    const history = await getImageRecognitionHistory({
      page: 1,
      size: 10,
      status: 'completed'
    });
    console.log('识别历史:', history);
  } catch (error) {
    console.error('获取历史失败:', error);
  }
};
```

## 五、测试场景

### 5.1 正常流程测试

1. ✅ 上传JPG格式图片 → 识别成功
2. ✅ 上传PNG格式图片 → 识别成功
3. ✅ 上传WEBP格式图片 → 识别成功
4. ✅ 获取识别历史 → 返回列表
5. ✅ 基于识别结果进行AI对话 → 返回流式响应

### 5.2 异常场景测试

1. ❌ 未登录上传图片 → 返回401
2. ❌ 上传不支持格式（如TXT） → 返回400
3. ❌ 上传超大文件（>10MB） → 返回400
4. ❌ 使用不存在的recognitionId → 返回错误
5. ❌ Qwen-VL API调用失败 → 状态更新为failed

### 5.3 边界测试

1. ✅ 上传空文件 → 返回400
2. ✅ 上传1MB图片 → 识别成功
3. ✅ 上传9.9MB图片 → 识别成功
4. ✅ 上传10.1MB图片 → 返回400
5. ✅ 连续上传多张图片 → 都能识别成功

## 七、常见问题

### Q1: 识别失败，状态为failed

**可能原因：**
- Qwen-VL API Key未配置或无效
- 网络连接问题
- API调用超时

**解决方法：**
1. 检查 `QWEN_VL_API_KEY` 环境变量
2. 查看后端日志中的错误信息
3. 检查网络连接和防火墙设置

### Q2: 上传图片后没有响应

**可能原因：**
- 文件大小超限
- 文件格式不支持
- 服务器存储空间不足

**解决方法：**
1. 检查图片大小（应<10MB）
2. 确认图片格式（jpg/jpeg/png/webp）
3. 检查 `./data/images` 目录权限

### Q3: 识别历史查询为空

**可能原因：**
- 用户还没有上传过图片
- 查询条件过滤掉了所有记录

**解决方法：**
1. 先上传一张图片进行识别
2. 检查查询参数（status、page、size）

### Q4: 基于图片的对话没有响应

**可能原因：**
- recognitionId不存在
- 识别状态不是completed
- SSE连接问题

**解决方法：**
1. 确认recognitionId正确
2. 检查识别状态是否为completed
3. 使用支持SSE的客户端测试
