# 媒体文件实体类设计文档

## 文档说明

本文档详细说明 `MediaFile`（媒体文件/知识文档）实体类的设计，包括字段定义、数据类型、约束条件、枚举类型、扩展字段等。

---

## 一、实体类概述

### 1.1 基本信息

| 属性 | 值 |
|------|-----|
| **实体类名** | `MediaFile` |
| **表名** | `media_files` |
| **说明** | 存储知识库文档和媒体文件信息，支持多种文件类型和访问方式 |
| **包路径** | `org.backend.cleanersupportagentbackend.entity` |

### 1.2 业务场景

`MediaFile` 实体用于存储系统中的媒体文件和知识文档，包括：

- **销售资料**：PDF、PPT、Excel 等销售相关文档
- **产品资料**：产品图片、视频、规格说明等
- **维护资料**：维护视频、操作指南、故障排除文档
- **培训资料**：培训视频、教程文档等

支持多种访问方式：
- **Seafile**：通过 NJU Box (Seafile) 访问
- **本地文件**：存储在服务器本地
- **对象存储**：存储在 OSS 等云存储服务

## 二、字段定义

### 2.1 完整字段列表

| 字段名 | Java类型 | 数据库类型 | 约束 | 说明 |
|--------|----------|------------|------|------|
| id | Long | BIGINT | PK, AUTO_INCREMENT | 数据库主键（自增） |
| fileId | String(50) | VARCHAR(50) | UNIQUE, NOT NULL | 业务ID，如 "KB001" |
| title | String(200) | VARCHAR(200) | NOT NULL | 文件标题 |
| type | FileType | VARCHAR(50) | NOT NULL | 文件类型（枚举） |
| category | String(100) | VARCHAR(100) | NULL | 分类，如 maintenance, sales, training |
| mediaUrl | String(500) | VARCHAR(500) | NULL | 媒体文件URL（视频、PDF等） |
| coverUrl | String(500) | VARCHAR(500) | NULL | 封面图URL |
| filePath | String(500) | VARCHAR(500) | NULL | 本地文件路径 |
| repoId | String(100) | VARCHAR(100) | NULL | Seafile 资料库ID |
| seafilePath | String(500) | VARCHAR(500) | NULL | Seafile 中的文件路径（完整路径） |
| downloadUrl | String(500) | VARCHAR(500) | NULL | 下载链接（临时链接，有时效性） |
| previewUrl | String(500) | VARCHAR(500) | NULL | 预览链接（Web预览地址） |
| isViewable | Boolean | BOOLEAN | NULL | 是否支持在线预览 |
| accessMethod | AccessMethod | VARCHAR(20) | NULL | 访问方式枚举 |
| createdAt | LocalDateTime | TIMESTAMP | NOT NULL | 创建时间（自动设置） |
| updatedAt | LocalDateTime | TIMESTAMP | NOT NULL | 更新时间（自动更新） |

### 2.2 字段详细说明

#### 2.2.1 基础字段

**id（数据库主键）**
- **类型**：`Long`
- **说明**：数据库自增主键，用于数据库内部关联
- **特点**：不暴露给前端，仅用于数据库操作

**fileId（业务ID）**
- **类型**：`String(50)`
- **约束**：`UNIQUE, NOT NULL`
- **说明**：业务ID，用于API接口和前端展示
- **格式**：`KB` + UUID前6位（大写），如 `KB123456`
- **生成规则**：见「六、业务ID生成规则」

**title（文件标题）**
- **类型**：`String(200)`
- **约束**：`NOT NULL`
- **说明**：文件标题，优先使用中文名称映射，否则使用文件名
- **示例**：`"市场竞争分析报告"`、`"Artist 1 用户手册（英文版）"`

#### 2.2.2 类型与分类字段

**type（文件类型）**
- **类型**：`FileType`（枚举）
- **约束**：`NOT NULL`
- **说明**：文件类型，用于区分不同的媒体类型
- **枚举值**：见「三、枚举类型」

**category（分类）**
- **类型**：`String(100)`
- **约束**：`NULL`
- **说明**：文件分类，用于分类管理和搜索
- **常见值**：
  - `sales`：销售资料
  - `company`：公司介绍
  - `product`：产品资料
  - `maintenance`：技术支持/维护
  - `training`：培训资料

#### 2.2.3 内容字段

**mediaUrl（媒体文件URL）**
- **类型**：`String(500)`
- **约束**：`NULL`
- **说明**：媒体文件URL，用于视频、PDF等文件的直接访问
- **示例**：`"https://cdn.example.com/video.mp4"`

**coverUrl（封面图URL）**
- **类型**：`String(500)`
- **约束**：`NULL`
- **说明**：封面图URL，用于视频、文章等内容的封面展示
- **示例**：`"https://cdn.example.com/cover.jpg"`

#### 2.2.4 路径与访问字段

**filePath（本地文件路径）**
- **类型**：`String(500)`
- **约束**：`NULL`
- **说明**：本地文件路径，当 `accessMethod = LOCAL` 时使用
- **示例**：`"./data/uploads/video.mp4"`

**repoId（Seafile 资料库ID）**
- **类型**：`String(100)`
- **约束**：`NULL`
- **说明**：Seafile 资料库ID，当 `accessMethod = SEAFILE` 时使用
- **格式**：UUID格式
- **示例**：`"9136e9de-f49e-45f6-9aff-9e3abe07a257"`

**seafilePath（Seafile 文件路径）**
- **类型**：`String(500)`
- **约束**：`NULL`
- **说明**：文件在 Seafile 中的完整路径，当 `accessMethod = SEAFILE` 时使用
- **格式**：以 `/` 开头的绝对路径
- **示例**：`"/00-Sales Kit/00-Competitors in the market.pdf"`

**downloadUrl（下载链接）**
- **类型**：`String(500)`
- **约束**：`NULL`
- **说明**：文件下载链接，临时链接，有时效性
- **注意**：对于 Seafile 文件，此字段可能为空，需要实时获取

**previewUrl（预览链接）**
- **类型**：`String(500)`
- **约束**：`NULL`
- **说明**：文件预览链接，用于在线预览
- **格式**：Seafile Web 预览地址或直接访问URL
- **示例**：`"https://box.nju.edu.cn/lib/9136e9de-f49e-45f6-9aff-9e3abe07a257/file/%2F00-Sales%20Kit%2F00-Competitors%20in%20the%20market.pdf"`

**isViewable（是否可预览）**
- **类型**：`Boolean`
- **约束**：`NULL`
- **说明**：是否支持在线预览
- **判断规则**：根据文件扩展名判断，支持预览的类型包括：
  - PDF：`.pdf`
  - 图片：`.jpg`, `.jpeg`, `.png`, `.gif`, `.bmp`, `.webp`, `.svg`
  - 视频：`.mp4`, `.mov`, `.avi`, `.mkv`, `.flv`, `.wmv`, `.webm`, `.ts`
  - 音频：`.mp3`, `.wav`, `.flac`, `.aac`, `.ogg`, `.m4a`
- **默认值**：根据文件类型自动判断

**accessMethod（访问方式）**
- **类型**：`AccessMethod`（枚举）
- **约束**：`NULL`
- **说明**：文件访问方式，用于区分不同的存储和访问方式
- **枚举值**：见「三、枚举类型」
- **使用场景**：
  - `SEAFILE`：文件存储在 Seafile，需要通过 Seafile API 获取访问链接
  - `LOCAL`：文件存储在服务器本地，通过 `filePath` 访问
  - `OSS`：文件存储在对象存储服务，通过 `mediaUrl` 访问

#### 2.2.5 时间戳字段

**createdAt（创建时间）**
- **类型**：`LocalDateTime`
- **约束**：`NOT NULL, updatable = false`
- **说明**：记录创建时间，自动设置，不可更新
- **设置方式**：通过 `@PrePersist` 生命周期回调自动设置

**updatedAt（更新时间）**
- **类型**：`LocalDateTime`
- **约束**：`NOT NULL`
- **说明**：记录最后更新时间，每次更新时自动更新
- **设置方式**：通过 `@PrePersist` 和 `@PreUpdate` 生命周期回调自动更新

## 三、枚举类型

### 3.1 FileType（文件类型枚举）

**说明**：用于标识文件的媒体类型

```java
public enum FileType {
    Article,   // 文章
    Video,     // 视频
    PDF,       // PDF文档
    Image,     // 图片
    Excel,     // Excel表格
    PPT        // PowerPoint演示文稿
}
```

**映射规则**：

| 扩展名 | FileType | 是否可预览 |
|--------|----------|------------|
| `.pdf` | `PDF` | ✅ 是 |
| `.jpg`, `.jpeg`, `.png`, `.gif`, `.bmp`, `.webp`, `.svg` | `Image` | ✅ 是 |
| `.mp4`, `.mov`, `.avi`, `.mkv`, `.flv`, `.wmv`, `.webm`, `.ts` | `Video` | ✅ 是 |
| `.xlsx`, `.xls` | `Excel` | ❌ 否 |
| `.pptx`, `.ppt` | `PPT` | ❌ 否 |

**注意**：
- 音频文件（`.mp3`, `.wav` 等）当前没有对应的枚举值，建议使用 `Video` 或未来添加 `Audio` 类型
- 文章类型（`Article`）通常用于 Markdown 格式的文本内容

### 3.2 AccessMethod（访问方式枚举）

**说明**：用于标识文件的存储和访问方式

```java
public enum AccessMethod {
    SEAFILE,    // 通过 Seafile 访问
    LOCAL,      // 本地文件
    OSS         // 对象存储
}
```

**使用说明**：

| AccessMethod | 使用字段 | 说明 |
|--------------|----------|------|
| `SEAFILE` | `repoId`, `seafilePath` | 文件存储在 Seafile，需要通过 API 获取访问链接 |
| `LOCAL` | `filePath` | 文件存储在服务器本地文件系统 |
| `OSS` | `mediaUrl` | 文件存储在对象存储服务（如阿里云OSS、腾讯云COS等） |

## 四、完整实体类代码

### 4.1 实体类实现

```java
package org.backend.cleanersupportagentbackend.entity;

import jakarta.persistence.*;
import lombok.AllArgsConstructor;
import lombok.Builder;
import lombok.Data;
import lombok.NoArgsConstructor;

import java.time.LocalDateTime;

/**
 * 媒体文件/知识文档实体
 */
@Entity
@Table(name = "media_files")
@Data
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class MediaFile {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(unique = true, nullable = false, length = 50)
    private String fileId; // 业务ID，如 KB001

    @Column(nullable = false, length = 200)
    private String title; // 文件标题

    @Column(nullable = false, length = 50)
    @Enumerated(EnumType.STRING)
    private FileType type; // Article, Video, PDF, Image, Excel, PPT

    @Column(length = 100)
    private String category; // 分类，如 maintenance, sales, training

    @Column(length = 500)
    private String mediaUrl; // 媒体文件URL（视频、PDF等）

    @Column(length = 500)
    private String coverUrl; // 封面图URL

    @Column(length = 500)
    private String filePath; // 本地文件路径

    // Seafile 相关字段
    @Column(length = 100)
    private String repoId; // Seafile 资料库ID

    @Column(length = 500)
    private String seafilePath; // Seafile 中的文件路径（完整路径）

    @Column(length = 500)
    private String downloadUrl; // 下载链接（临时链接，有时效性）

    @Column(length = 500)
    private String previewUrl; // 预览链接（Web预览地址）

    @Column
    private Boolean isViewable; // 是否支持在线预览

    @Column(length = 20)
    @Enumerated(EnumType.STRING)
    private AccessMethod accessMethod; // 访问方式

    @Column(nullable = false, updatable = false)
    private LocalDateTime createdAt;

    @Column(nullable = false)
    private LocalDateTime updatedAt;

    @PrePersist
    protected void onCreate() {
        createdAt = LocalDateTime.now();
        updatedAt = LocalDateTime.now();
    }

    @PreUpdate
    protected void onUpdate() {
        updatedAt = LocalDateTime.now();
    }

    /**
     * 文件类型枚举
     */
    public enum FileType {
        Article,   // 文章
        Video,     // 视频
        PDF,       // PDF文档
        Image,     // 图片
        Excel,     // Excel表格
        PPT        // PowerPoint演示文稿
    }

    /**
     * 访问方式枚举
     */
    public enum AccessMethod {
        SEAFILE,    // 通过 Seafile 访问
        LOCAL,      // 本地文件
        OSS         // 对象存储
    }
}
```

### 4.2 注解说明

| 注解 | 说明 |
|------|------|
| `@Entity` | 标识为 JPA 实体类 |
| `@Table(name = "media_files")` | 指定数据库表名 |
| `@Id` | 标识主键字段 |
| `@GeneratedValue(strategy = GenerationType.IDENTITY)` | 主键自增策略 |
| `@Column` | 指定列属性（长度、是否可空等） |
| `@Enumerated(EnumType.STRING)` | 枚举存储为字符串 |
| `@PrePersist` | 保存前回调，用于设置创建时间 |
| `@PreUpdate` | 更新前回调，用于更新更新时间 |

## 五、业务ID生成规则

### 5.1 fileId 生成规则

**格式**：`KB` + UUID前6位（大写）

**示例**：`KB123456`、`KBABCDEF`

**生成方法**：

```java
public static String generateFileId() {
    String uuid = UUID.randomUUID().toString().replace("-", "");
    String suffix = uuid.substring(0, 6).toUpperCase();
    return "KB" + suffix;
}
```

**特点**：
- 全局唯一（基于 UUID）
- 长度固定（8个字符）
- 可读性好（以 `KB` 开头，表示 Knowledge Base）
- 不暴露数据库内部结构

## 六、数据约束与索引

### 6.1 唯一性约束

| 字段 | 约束类型 | 说明 |
|------|----------|------|
| `fileId` | UNIQUE | 业务ID唯一，用于API接口标识 |

### 6.2 非空约束

| 字段 | 约束 | 说明 |
|------|------|------|
| `fileId` | NOT NULL | 业务ID必填 |
| `title` | NOT NULL | 文件标题必填 |
| `type` | NOT NULL | 文件类型必填 |
| `createdAt` | NOT NULL | 创建时间必填 |
| `updatedAt` | NOT NULL | 更新时间必填 |

### 6.3 索引建议

**主键索引**：
- `id`：自动创建主键索引

**唯一索引**：
- `fileId`：用于快速查找文件

**普通索引**（建议添加）：
- `category`：用于分类查询
- `type`：用于类型筛选
- `accessMethod`：用于访问方式筛选
- `createdAt`：用于时间排序
- `(category, type)`：组合索引，用于分类和类型联合查询

**索引创建SQL示例**：

```sql
-- 分类索引
CREATE INDEX idx_media_files_category ON media_files(category);

-- 类型索引
CREATE INDEX idx_media_files_type ON media_files(type);

-- 访问方式索引
CREATE INDEX idx_media_files_access_method ON media_files(access_method);

-- 创建时间索引（用于排序）
CREATE INDEX idx_media_files_created_at ON media_files(created_at DESC);

-- 组合索引（分类+类型）
CREATE INDEX idx_media_files_category_type ON media_files(category, type);
```

## 七、使用示例

### 7.1 创建 Seafile 文件实体

```java
MediaFile file = MediaFile.builder()
    .fileId("KB123456")
    .title("市场竞争分析报告")
    .type(MediaFile.FileType.PDF)
    .category("sales")
    .repoId("9136e9de-f49e-45f6-9aff-9e3abe07a257")
    .seafilePath("/00-Sales Kit/00-Competitors in the market.pdf")
    .isViewable(true)
    .accessMethod(MediaFile.AccessMethod.SEAFILE)
    .build();

mediaFileRepository.save(file);
```

### 7.2 创建本地文件实体

```java
MediaFile file = MediaFile.builder()
    .fileId("KB789ABC")
    .title("维护视频教程")
    .type(MediaFile.FileType.Video)
    .category("maintenance")
    .filePath("./data/uploads/maintenance_video.mp4")
    .mediaUrl("/api/media/files/KB789ABC/video")
    .coverUrl("/api/media/files/KB789ABC/cover.jpg")
    .isViewable(true)
    .accessMethod(MediaFile.AccessMethod.LOCAL)
    .build();

mediaFileRepository.save(file);
```

### 7.3 查询示例

```java
// 根据 fileId 查询
Optional<MediaFile> file = mediaFileRepository.findByFileId("KB123456");

// 根据分类查询
List<MediaFile> salesFiles = mediaFileRepository.findByCategory("sales");

// 组合查询（分类+关键词）
List<MediaFile> results = mediaFileRepository.searchByCategoryAndQuery(
    "maintenance", "传感器"
);

// 查询 Seafile 文件
@Query("SELECT m FROM MediaFile m WHERE m.accessMethod = 'SEAFILE'")
List<MediaFile> findSeafileFiles();
```

## 八、字段使用场景总结

### 8.1 不同访问方式的字段使用

| 访问方式 | 必填字段 | 可选字段 | 说明 |
|----------|----------|----------|------|
| `SEAFILE` | `repoId`, `seafilePath` | `downloadUrl`, `previewUrl`, `isViewable` | Seafile 文件，通过 API 获取链接 |
| `LOCAL` | `filePath` | `mediaUrl` | 本地文件，通过文件路径访问 |
| `OSS` | `mediaUrl` | `coverUrl` | 对象存储，通过 URL 直接访问 |

### 8.2 不同文件类型的字段使用

| 文件类型 | 常用字段 | 说明 |
|----------|----------|------|
| `Video` | `mediaUrl`, `coverUrl` | 视频类，需要媒体URL和封面 |
| `PDF` | `mediaUrl`, `previewUrl` | PDF文档，支持在线预览 |
| `Image` | `mediaUrl`, `coverUrl` | 图片类，媒体URL和封面可能相同 |
| `Excel` | `mediaUrl`, `downloadUrl` | Excel表格，通常需要下载 |
| `PPT` | `mediaUrl`, `downloadUrl` | PPT演示文稿，通常需要下载 |

## 九、注意事项

### 9.1 字段互斥性

- **路径字段互斥**：
  - `seafilePath`（Seafile 文件）与 `filePath`（本地文件）互斥
  - 根据 `accessMethod` 决定使用哪个字段

- **URL字段关系**：
  - `mediaUrl`：直接访问URL（适用于 OSS 或本地文件）
  - `downloadUrl`：下载链接（临时链接，有时效性）
  - `previewUrl`：预览链接（适用于可预览的文件）

### 9.2 数据一致性

- **isViewable 自动判断**：建议在保存时根据文件类型自动设置 `isViewable`
- **previewUrl 自动生成**：对于 Seafile 文件，`previewUrl` 可以根据 `repoId` 和 `seafilePath` 自动生成
- **downloadUrl 实时获取**：对于 Seafile 文件，`downloadUrl` 应该实时从 Seafile API 获取，不存储在数据库中

### 9.3 性能优化

- **索引优化**：为常用查询字段添加索引（`category`, `type`, `accessMethod`）
- **分页查询**：列表查询建议使用分页，避免一次性加载大量数据

## 十、数据库表结构

### 10.1 主表结构（media_files）

```sql
CREATE TABLE media_files (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    file_id VARCHAR(50) UNIQUE NOT NULL,
    title VARCHAR(200) NOT NULL,
    type VARCHAR(50) NOT NULL,
    category VARCHAR(100),
    media_url VARCHAR(500),
    cover_url VARCHAR(500),
    file_path VARCHAR(500),
    repo_id VARCHAR(100),
    seafile_path VARCHAR(500),
    download_url VARCHAR(500),
    preview_url VARCHAR(500),
    is_viewable BOOLEAN,
    access_method VARCHAR(20),
    created_at TIMESTAMP NOT NULL,
    updated_at TIMESTAMP NOT NULL,
    
    INDEX idx_file_id (file_id),
    INDEX idx_category (category),
    INDEX idx_type (type),
    INDEX idx_access_method (access_method),
    INDEX idx_created_at (created_at DESC)
);
```
