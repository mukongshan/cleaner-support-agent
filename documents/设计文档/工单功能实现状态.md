### 2.2 需要新增的接口

#### 2.2.1 工单对话接口

**接口1：获取工单对话**
- **路径：** `GET /tickets/{ticketId}/conversation`
- **功能：** 获取工单关联的对话信息（工单专用对话，与AI对话无关）
- **权限：** 工单创建者、分配的工程师、管理员

**接口2：在工单对话中发送消息**
- **路径：** `POST /tickets/{ticketId}/conversation/messages`
- **功能：** 在工单对话中发送消息
- **请求参数：**
  ```json
  {
    "content": "消息内容"
  }
  ```
- **权限：** 工单创建者、分配的工程师、管理员
- **说明：** 自动根据当前用户角色设置message role（user或engineer）

#### 2.2.2 工单反馈接口

**接口：提交工单反馈**
- **路径：** `POST /tickets/{ticketId}/feedback`
- **功能：** 用户对已完成工单提交反馈
- **请求参数：**
  ```json
  {
    "type": "like",  // like 或 dislike
    "comment": "反馈意见"
  }
  ```
- **权限：** 工单创建者
- **说明：** 只有工单状态为completed时才能提交反馈

#### 2.2.3 工单统计接口（可选，扩展功能）

**接口：获取工单统计数据**
- **路径：** `GET /tickets/statistics`
- **功能：** 获取工单统计数据（管理员）
- **查询参数：**
  - `startDate` (可选) - 开始日期
  - `endDate` (可选) - 结束日期
  - `groupBy` (可选) - 分组方式：status/priority/type
- **权限：** 管理员

### 2.3 需要修改的现有功能

#### 2.3.1 创建工单接口

**需要修改：**
1. 修改字段名：`relatedChatId` → `relatedAiChatId`
2. **新增功能：** 工单创建成功后，自动创建工单对话（Conversation，类型为TICKET_CONVERSATION）
3. 建立Ticket和Conversation的双向关联

**修改后的流程：**
```java
1. 创建Ticket实体
2. 保存Ticket
3. 创建Conversation（type = TICKET_CONVERSATION）
4. 建立双向关联：ticket.conversation = conversation; conversation.ticket = ticket
5. 保存Conversation
```

#### 2.3.2 更新工单状态接口

**需要修改：**
1. 支持分派工程师（设置engineerName和engineerId）
2. 支持设置预计处理时间（estimatedTime）

**新增请求参数：**
```json
{
  "status": "processing",
  "engineerName": "李师傅",
  "engineerId": "U12345",
  "estimatedTime": "2小时内",
  "comments": "备注信息"
}
```

## 7. 数据库迁移脚本

### 7.1 Ticket表修改

```sql
-- 修改字段名
ALTER TABLE tickets 
CHANGE COLUMN related_chat_id related_ai_chat_id VARCHAR(100);

-- 新增字段
ALTER TABLE tickets 
ADD COLUMN conversation_id BIGINT,
ADD COLUMN engineer_id VARCHAR(50),
ADD FOREIGN KEY (conversation_id) REFERENCES conversations(id);

CREATE INDEX idx_ticket_conversation ON tickets(conversation_id);
```

### 7.2 Conversation表扩展

```sql
-- 新增字段
ALTER TABLE conversations 
ADD COLUMN type VARCHAR(50) DEFAULT 'AI_CONVERSATION',
ADD COLUMN ticket_id BIGINT,
ADD FOREIGN KEY (ticket_id) REFERENCES tickets(id);

CREATE INDEX idx_conversation_type ON conversations(type);
CREATE INDEX idx_conversation_ticket ON conversations(ticket_id);
```
