# 图片识别功能设计文档

## 文档说明

本文档详细说明清洁机器人智能指南平台的图片识别功能设计，包括业务流程、技术架构、实体设计、API接口、服务层实现等。

---

## 一、功能概述

### 1.1 业务场景

用户在使用AI助手时，可以上传机器人故障相关的图片，系统会：
1. **图片识别**：使用Qwen-VL模型将图片转换为文字描述
2. **智能问答**：基于图片描述，结合用户问题，向LLM（Dify）提问
3. **结果展示**：返回AI的回复和建议

### 1.2 核心价值

- **降低沟通成本**：用户无需详细描述故障，拍照即可
- **提高诊断准确性**：AI可以"看到"实际故障情况
- **提升用户体验**：支持多模态交互，更符合用户习惯

## 四、实体类设计

### 4.1 ImageRecognition（图片识别记录实体）

**表名：** `image_recognitions`

**说明：** 存储用户上传的图片及其识别结果

#### 字段定义

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | Long | PK, AUTO_INCREMENT | 数据库主键 |
| recognitionId | String(50) | UNIQUE, NOT NULL | 业务ID，如 "IMG20240120001" |
| user | User | FK, NOT NULL | 关联用户（多对一关系） |
| imageUrl | String(500) | NOT NULL | 图片URL |
| imagePath | String(500) | NOT NULL | 本地文件路径 |
| description | String(TEXT) | NULL | 图片描述（Qwen-VL识别结果） |
| status | RecognitionStatus | NOT NULL | 识别状态（枚举） |
| errorMessage | String(500) | NULL | 错误信息（识别失败时） |
| createdAt | LocalDateTime | NOT NULL | 创建时间 |
| updatedAt | LocalDateTime | NOT NULL | 更新时间 |

#### 枚举类型

```java
public enum RecognitionStatus {
    pending,      // 待识别
    processing,   // 识别中
    completed,    // 识别完成
    failed        // 识别失败
}
```

#### 关系定义

- **多对一（Many-to-One）**：多个识别记录属于一个用户
  ```java
  @ManyToOne(fetch = FetchType.LAZY)
  @JoinColumn(name = "user_id", nullable = false)
  private User user;
  ```

#### 特殊设计

- **业务ID格式**：`IMG` + 日期(8位) + 序号(3位)
- **状态流转**：pending → processing → completed/failed
- **错误处理**：识别失败时保存错误信息，便于排查

## 五、API接口设计

### 5.1 上传图片并识别

**接口路径**：`POST /api/cleaner-support/v2/image-reco`

**是否需要认证**：是

**Content-Type**：`multipart/form-data`

**请求参数**：

- `image`：图片文件（MultipartFile）
  - 支持格式：JPG, PNG, JPEG, WEBP
  - 最大大小：10MB

**响应示例**：
```json
{
  "code": 200,
  "message": "识别成功",
  "data": {
    "recognitionId": "IMG20240120001",
    "imageUrl": "/api/cleaner-support/v2/media/images/xxx.jpg",
    "description": "图片中显示一个清洁机器人，主刷部分被地毯缠绕，机器人显示错误代码E04...",
    "status": "completed"
  }
}
```

**错误响应**：
```json
{
  "code": 500,
  "message": "图片识别失败：API调用超时",
  "data": null
}
```

### 5.2 基于图片识别结果进行AI对话

**接口路径**：`POST /api/cleaner-support/v2/ai/chat/with-image`

**是否需要认证**：是

**Content-Type**：`application/json`

**请求参数**：
```json
{
  "recognitionId": "IMG20240120001",  // 图片识别记录ID（必填）
  "query": "这个问题怎么解决？",        // 用户问题（可选，如果不提供则只基于图片描述）
  "conversationId": "conv_12345678"   // 会话ID（可选，用于关联历史会话）
}
```

**响应格式**：SSE流式响应（与普通对话接口相同）

**响应示例**：
```
data: {"event": "message", "answer": "根据图片描述，您的机器人主刷被地毯缠绕...", "conversation_id": "conv_12345678"}
data: {"event": "message_end", "metadata": {}}
```

### 5.3 获取图片识别历史

**接口路径**：`GET /api/cleaner-support/v2/image-reco/history`

**是否需要认证**：是

**查询参数**（可选）：
- `status`：识别状态过滤（pending, processing, completed, failed）
- `page`：页码（默认1）
- `size`：每页数量（默认10）

**响应示例**：
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "total": 25,
    "page": 1,
    "size": 10,
    "items": [
      {
        "recognitionId": "IMG20240120001",
        "imageUrl": "/api/cleaner-support/v2/media/images/xxx.jpg",
        "description": "图片中显示一个清洁机器人...",
        "status": "completed",
        "createdAt": "2024-01-20T10:30:00"
      }
    ]
  }
}
```

## 六、服务层设计

### 6.1 ImageRecognitionService（图片识别服务）

**职责**：
- 保存上传的图片
- 调用Qwen-VL API进行图片识别
- 管理识别状态
- 查询识别历史

**核心方法**：

```java
@Service
public class ImageRecognitionService {
    
    /**
     * 上传图片并识别
     */
    @Transactional
    public ImageRecognitionResponse recognizeImage(String userId, MultipartFile image);
    
    /**
     * 根据ID获取识别结果
     */
    public ImageRecognition getRecognitionById(String recognitionId);
    
    /**
     * 获取用户的识别历史
     */
    public Page<ImageRecognition> getUserRecognitions(String userId, String status, Pageable pageable);
    
    /**
     * 调用Qwen-VL API识别图片
     */
    private String callQwenVLAPI(byte[] imageBytes);
}
```

### 6.2 QwenVLClient（Qwen-VL API客户端）

**职责**：
- 封装Qwen-VL API调用
- 处理API请求和响应
- 错误处理和重试

**核心方法**：

```java
@Component
public class QwenVLClient {
    
    /**
     * 调用视觉识别API
     */
    public String recognizeImage(byte[] imageBytes, String imageFormat);
    
    /**
     * 构建API请求
     */
    private HttpEntity<MultiValueMap<String, Object>> buildRequest(byte[] imageBytes);
    
    /**
     * 解析API响应
     */
    private String parseResponse(String responseBody);
}
```

### 6.3 AiService扩展（AI对话服务扩展）

**职责**：
- 集成图片描述到对话流程

### 8.2 错误处理

| HTTP状态码 | 错误类型 | 处理方式 |
|------------|----------|----------|
| 400 | 请求参数错误 | 返回错误信息，不重试 |
| 401 | API密钥无效 | 记录日志，返回错误 |
| 429 | 请求频率限制 | 等待后重试（指数退避） |
| 500 | 服务器错误 | 重试（最多3次） |
| 超时 | 网络超时 | 重试（最多3次） |

**基础提示词**：
```
请详细描述这张图片中的内容，特别是与清洁机器人相关的故障或问题。
包括：
1. 机器人的状态和外观
2. 可见的错误代码或指示灯
3. 可能存在的故障或异常情况
4. 任何与清洁、维护相关的细节
```

**优化提示词**（针对故障诊断）：
```
你是一位专业的清洁机器人维修专家。请仔细观察这张图片，详细描述：

1. **机器人状态**：机器人的工作状态（运行中/停止/报错）
2. **错误信息**：屏幕上显示的错误代码、指示灯状态
3. **故障现象**：可见的故障或异常（如缠绕、堵塞、损坏等）
4. **环境因素**：周围环境可能影响机器人的因素
5. **建议关注点**：需要特别注意的细节

请用专业但易懂的语言描述，为后续的故障诊断提供准确信息。
```

## 十二、性能优化

### 12.1 异步处理（可选）

**场景**：图片识别耗时较长（3-10秒）

**方案**：
1. 上传图片后立即返回，状态为`pending`
2. 异步调用Qwen-VL API
3. 识别完成后更新状态为`completed`
4. 前端轮询或使用WebSocket推送结果

**实现**：
```java
@Async
public CompletableFuture<String> recognizeImageAsync(String recognitionId);
```

### 12.2 结果缓存

**场景**：相同图片可能被多次识别

**方案**：
- 使用图片MD5作为缓存key
- 缓存识别结果（TTL：1小时）
- 相同图片直接返回缓存结果

### 12.3 图片压缩

**场景**：减少API调用时间和成本

**方案**：
- 上传后自动压缩图片（保持质量）
- 压缩到合适尺寸（如1920x1080）
- 减少base64编码后的数据量

## 十三、安全考虑

### 13.1 图片验证

- **格式验证**：只允许图片格式（jpg, png等）
- **大小限制**：最大10MB
- **内容验证**：检查文件头，防止伪造扩展名
- **病毒扫描**：生产环境建议集成病毒扫描

### 13.2 访问控制

- **用户隔离**：用户只能访问自己的图片识别记录
- **URL签名**：图片访问URL使用签名，防止未授权访问
- **API限流**：限制单个用户的识别频率（如每分钟5次）

### 13.3 数据隐私

- **敏感信息过滤**：识别结果中过滤敏感信息（如个人信息）
- **数据保留**：设置数据保留期限（如90天）
- **删除机制**：用户可删除自己的识别记录

### 14.9 常见问题

**Q1: 识别失败，状态为failed**

**可能原因：**
- Qwen-VL API Key未配置或无效
- 网络连接问题
- API调用超时

**解决方法：**
1. 检查 `QWEN_VL_API_KEY` 环境变量
2. 查看后端日志中的错误信息
3. 检查网络连接和防火墙设置

**Q2: 上传图片后没有响应**

**可能原因：**
- 文件大小超限
- 文件格式不支持
- 服务器存储空间不足

**解决方法：**
1. 检查图片大小（应<10MB）
2. 确认图片格式（jpg/jpeg/png/webp）
3. 检查 `./data/images` 目录权限

**Q3: 识别历史查询为空**

**可能原因：**
- 用户还没有上传过图片
- 查询条件过滤掉了所有记录

**解决方法：**
1. 先上传一张图片进行识别
2. 检查查询参数（status、page、size）

**Q4: 基于图片的对话没有响应**

**可能原因：**
- recognitionId不存在
- 识别状态不是completed
- SSE连接问题

**解决方法：**
1. 确认recognitionId正确
2. 检查识别状态是否为completed
3. 使用支持SSE的客户端测试
