# 控制类设计文档

## 1. 概述

控制层（Controller）是系统的API接口层，负责接收HTTP请求、参数验证、调用服务层处理业务逻辑，并返回统一格式的响应数据。

### 1.1 设计原则

- **统一响应格式**：所有接口使用 `ApiResponse<T>` 统一包装响应数据
- **统一认证管理**：通过 `AuthInterceptor` 拦截器统一处理登录验证
- **参数自动注入**：使用 `@CurrentUserId` 注解自动注入当前登录用户ID
- **异常统一处理**：通过 `GlobalExceptionHandler` 统一处理异常

### 1.2 基础路径

所有API接口的基础路径：`/api/cleaner-support/v2`

### 1.3 统一响应格式

```json
{
  "code": 200,          // 状态码：200-成功, 400-业务错误, 401-未登录, 500-服务器错误
  "message": "操作成功", // 提示信息
  "data": {}            // 具体业务数据（泛型T）
}
```

### 1.4 认证机制

- **请求头**：`Authorization: Bearer <token>`
- **自动注入**：使用 `@CurrentUserId String userId` 自动获取当前登录用户ID
- **公开接口**：登录接口、媒体文件列表/详情接口无需认证

## 2. 用户认证接口 (AuthController)

**路径前缀**：`/api/cleaner-support/v2/users`

### 2.1 用户登录

**接口路径**：`POST /users/login`

**是否需要认证**：否（公开接口）

**请求参数**：
```json
{
  "username": "13800138000",  // 手机号
  "password": "hashed_password", // 密码
  "loginType": "sms"          // 登录类型：password 或 sms
}
```

**响应示例**：
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "token": "mock_token_U12345",
    "userId": "U12345",
    "nickname": "张先生",
    "avatar": "https://cdn.com/avatar.jpg"
  }
}
```

**错误响应**：
```json
{
  "code": 400,
  "message": "用户名或密码错误",
  "data": null
}
```

## 3. AI智能问答接口 (AiController)

**路径前缀**：`/api/cleaner-support/v2/ai`

### 3.1 发送AI对话消息（流式响应）

**接口路径**：`POST /ai/chat`

**是否需要认证**：是

**Content-Type**：`application/json`

**响应类型**：`text/event-stream` (SSE流式响应)

**请求参数**：
```json
{
  "query": "主刷卷入地毯了怎么办？",
  "conversationId": "conv_12345678",  // 可选，用于关联历史会话
  "deviceInfo": {                      // 可选
    "model": "X10 Pro",
    "errorCode": "E04"
  }
}
```

**响应格式（SSE）**：
```
data: {"event":"message","answer":"检测到您的机器人报...","conversation_id":"conv_12345678"}

data: {"event":"message_end","metadata":{}}
```

**说明**：
- 使用 Server-Sent Events (SSE) 实现流式响应
- 后端作为中转，将Dify AI的响应流式转发给前端
- 自动保存用户消息和AI回复到数据库

### 3.2 获取历史会话列表

**接口路径**：`GET /ai/conversations`

**是否需要认证**：是

**响应示例**：
```json
{

  "message": "操作成功",
  "data": [
    {
      "id": "conv_12345678",
      "title": "如何清理主刷？",

}
```

### 3.3 获取会话详情

**接口路径**：`GET /ai/conversations/{conversationId}`

**路径参数**：

## 6. 错误码说明

| 状态码 | 说明 | 示例场景 |
|--------|------|----------|
| 200 | 成功 | 请求处理成功 |
| 400 | 业务错误 | 参数验证失败、业务逻辑错误 |
| 401 | 未授权 | 未登录或token无效 |
| 404 | 资源不存在 | 查询的ID不存在 |
| 500 | 服务器错误 | 系统内部异常 |
| 501 | 功能未实现 | 预留接口，功能待开发 |

## 7. 认证拦截器说明

### 7.1 工作原理

1. **拦截路径**：所有 `/api/**` 路径（除公开接口外）
2. **验证流程**：
   - 从 `Authorization` header 提取 token
   - 使用 `TokenUtil` 解析 token 获取 userId
   - 将 userId 存入 request attribute
3. **自动注入**：Controller 方法使用 `@CurrentUserId String userId` 自动获取

### 7.2 公开接口（无需认证）

- `/api/cleaner-support/v2/users/login` - 登录接口
- `/api/cleaner-support/v2/media/files` - 媒体文件列表
- `/api/cleaner-support/v2/media/files/**` - 媒体文件详情
- `/mcp/**` - MCP协议接口

### 7.3 未登录响应

```json
{
  "code": 401,
  "message": "请先登录",
  "data": null
}
```

## 8. 接口调用示例

### 8.1 完整调用流程

```javascript
// 1. 用户登录
const loginResponse = await fetch('/api/cleaner-support/v2/users/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    username: '13800138000',
    password: 'password123',
    loginType: 'password'
  })
});
const { data: { token } } = await loginResponse.json();

// 2. 使用token调用需要认证的接口
const profileResponse = await fetch('/api/cleaner-support/v2/users/profile', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
});
const profile = await profileResponse.json();

// 3. AI对话（SSE流式）
const eventSource = new EventSource('/api/cleaner-support/v2/ai/chat', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
});
eventSource.onmessage = (event) => {
  const data = JSON.parse(event.data);
  console.log(data.answer);
};
```

## 9. 注意事项

1. **时间格式**：所有时间字段使用 ISO 8601 格式，如 `2024-01-20T14:30:00`
2. **分页**：当前版本未实现分页，后续版本可添加
3. **文件大小限制**：上传文件最大 20MB（可在配置文件中调整）
4. **Token有效期**：当前使用mock token，实际JWT token需要设置有效期
5. **跨域**：已配置CORS，支持前端跨域访问
