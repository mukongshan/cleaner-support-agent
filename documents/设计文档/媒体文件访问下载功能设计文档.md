# 媒体文件设计文档

## 文档说明

本文档详细说明如何将 NJU Box (Seafile) 中的媒体文件集成到系统中，包括数据转换、访问与下载实现、接口暴露等。

---

## 一、需求概述

### 1.1 背景

系统需要集成 NJU Box (Seafile) 中的媒体文件，包括：
- 销售资料（PDF、PPT、Excel）
- 产品图片和视频
- 维护视频
- 产品证书文档

这些文件存储在 NJU Box 服务器上，需要通过 API 获取访问链接，并支持：
- 在线预览（支持的文件类型）
- 文件下载
- 文件列表浏览

### 1.2 数据来源

- **文件路径列表**：`res/file_path.py` 中的 `file_paths` 数组
- **中文名称映射**：`res/file_path.py` 中的 `chinese_to_path_mapping` 字典
- **访问方式**：基于 `res/文件浏览与下载.py` 的实现逻辑

### 1.3 技术要点

- **服务器地址**：`https://box.nju.edu.cn`
- **认证方式**：使用 Repository Token（`REPO_TOKEN`）
- **资料库ID**：`REPO_ID`（UUID格式）
- **API端点**：`/api/v2.1/via-repo-token/download-link/`
- **Web预览**：`/lib/{repo_id}/file/{encoded_path}`

### 2.2 文件类型映射规则

根据文件扩展名，自动识别 `FileType`：

| 扩展名 | FileType | 是否可预览 |
|--------|----------|------------|

| .mp4, .mov, .avi, .mkv, .flv, .wmv, .webm, .ts | Video | ✅ 是 |

## 三、数据转换实现

### 3.1 文件路径到实体转换

#### 3.1.1 转换流程

1. **读取文件路径列表**：从 `file_path.py` 读取 `file_paths` 数组
2. **解析文件信息**：
   - 提取文件名（作为 `title`）
   - 提取文件扩展名（确定 `type`）
   - 提取目录路径（确定 `category`）
   - 从 `chinese_to_path_mapping` 获取中文标题（优先使用）
3. **生成实体对象**：
   - 生成 `fileId`（格式：`KB` + UUID前6位）
   - 设置 `seafilePath`（完整路径）
   - 设置 `repoId`（配置的 REPO_ID）
   - 设置 `accessMethod = AccessMethod.SEAFILE`
   - 判断 `isViewable`

#### 3.1.2 分类规则

根据文件路径前缀，自动分类：

| 路径前缀 | category | 说明 |
|----------|----------|------|
| `/00-Sales Kit/` | `sales` | 销售资料 |
| `/01-Presale-Company/` | `company` | 公司介绍 |
| `/02-Presale-Product/` | `product` | 产品资料 |
| `/04-Technical Support/` | `maintenance` | 技术支持/维护 |

#### 3.1.3 代码实现示例

```java
@Service
public class MediaFileImportService {
    
    @Value("${seafile.repo-id}")
    private String repoId;
    
    @Value("${seafile.server-url}")
    private String serverUrl;
    
    /**
     * 从文件路径列表批量导入媒体文件
     */
    @Transactional
    public void importFromFilePaths(List<String> filePaths, 
                                    Map<String, String> chineseMapping) {
        List<MediaFile> mediaFiles = new ArrayList<>();
        
        for (String filePath : filePaths) {
            MediaFile mediaFile = convertToMediaFile(filePath, chineseMapping);
            mediaFiles.add(mediaFile);
        }
        
        mediaFileRepository.saveAll(mediaFiles);
    }
    
    /**
     * 将文件路径转换为 MediaFile 实体
     */
    private MediaFile convertToMediaFile(String filePath, 
                                         Map<String, String> chineseMapping) {
        // 1. 提取文件名和扩展名
        String fileName = extractFileName(filePath);
        String extension = extractExtension(filePath);
        
        // 2. 确定文件类型
        FileType fileType = determineFileType(extension);
        
        // 3. 确定分类
        String category = determineCategory(filePath);
        
        // 4. 获取标题（优先使用中文映射）
        String title = chineseMapping.getOrDefault(filePath, fileName);
        
        // 5. 判断是否可预览
        boolean isViewable = isViewableFile(extension);
        
        // 6. 生成预览URL
        String previewUrl = generatePreviewUrl(repoId, filePath);
        
        // 7. 构建实体
        return MediaFile.builder()
                .fileId(generateFileId())
                .title(title)
                .type(fileType)
                .category(category)
                .seafilePath(filePath)
                .repoId(repoId)
                .previewUrl(previewUrl)
                .isViewable(isViewable)
                .accessMethod(AccessMethod.SEAFILE)
                .build();
    }
    
    /**
     * 生成预览URL
     */
    private String generatePreviewUrl(String repoId, String filePath) {
        try {
            String encodedPath = URLEncoder.encode(filePath, StandardCharsets.UTF_8);
            return String.format("%s/lib/%s/file/%s", serverUrl, repoId, encodedPath);
        } catch (Exception e) {
            return null;
        }
    }
    
    /**
     * 判断文件是否可在线预览
     */
    private boolean isViewableFile(String extension) {
        Set<String> viewableExtensions = Set.of(
            ".pdf", ".jpg", ".jpeg", ".png", ".gif", ".bmp", ".webp", ".svg",
            ".mp4", ".mov", ".avi", ".mkv", ".flv", ".wmv", ".webm", ".ts",
            ".mp3", ".wav", ".flac", ".aac", ".ogg", ".m4a"
        );
        return viewableExtensions.contains(extension.toLowerCase());
    }
    
    // ... 其他辅助方法
}
```

## 四、访问与下载实现

### 4.1 Seafile 服务封装

创建一个 `SeafileService` 来封装与 Seafile API 的交互：

```java
@Service
public class SeafileService {
    
    @Value("${seafile.server-url}")
    private String serverUrl;
    
    @Value("${seafile.repo-token}")
    private String repoToken;
    
    @Value("${seafile.repo-id}")
    private String repoId;
    
    @Value("${seafile.repo-password:}")
    private String repoPassword;
    
    private final RestTemplate restTemplate;
    
    /**
     * 获取文件下载链接
     * 
     * @param filePath 文件在 Seafile 中的路径
     * @return 下载链接（临时链接，有时效性）
     */
    public String getDownloadLink(String filePath) {
        String url = serverUrl + "/api/v2.1/via-repo-token/download-link/";
        
        HttpHeaders headers = new HttpHeaders();
        headers.set("Authorization", "Bearer " + repoToken);
        
        MultiValueMap<String, String> params = new LinkedMultiValueMap<>();
        params.add("repo_id", repoId);
        params.add("path", filePath);
        if (StringUtils.hasText(repoPassword)) {
            params.add("password", repoPassword);
        }
        
        HttpEntity<?> entity = new HttpEntity<>(headers);
        UriComponentsBuilder builder = UriComponentsBuilder.fromHttpUrl(url)
                .queryParams(params);
        
        try {
            ResponseEntity<String> response = restTemplate.exchange(
                    builder.toUriString(),
                    HttpMethod.GET,
                    entity,
                    String.class
            );
            
            if (response.getStatusCode().is2xxSuccessful()) {
                String downloadUrl = response.getBody();
                // 清理响应中的引号
                if (downloadUrl != null) {
                    downloadUrl = downloadUrl.trim().replaceAll("^[\"']|[\"']$", "");
                }
                return downloadUrl;
            }
        } catch (Exception e) {
            throw new RuntimeException("获取下载链接失败: " + e.getMessage(), e);
        }
        
        return null;
    }
    
    /**
     * 生成 Web 预览链接
     * 
     * @param filePath 文件在 Seafile 中的路径
     * @return 预览链接
     */
    public String generatePreviewUrl(String filePath) {
        try {
            String encodedPath = URLEncoder.encode(filePath, StandardCharsets.UTF_8);
            return String.format("%s/lib/%s/file/%s", serverUrl, repoId, encodedPath);
        } catch (Exception e) {
            throw new RuntimeException("生成预览链接失败: " + e.getMessage(), e);
        }
    }
    
    /**
     * 判断文件是否支持在线预览
     */
    public boolean isViewableFile(String filePath) {
        String extension = extractExtension(filePath);
        Set<String> viewableExtensions = Set.of(
            ".pdf", ".jpg", ".jpeg", ".png", ".gif", ".bmp", ".webp", ".svg",
            ".mp4", ".mov", ".avi", ".mkv", ".flv", ".wmv", ".webm", ".ts",
            ".mp3", ".wav", ".flac", ".aac", ".ogg", ".m4a"
        );
        return viewableExtensions.contains(extension.toLowerCase());
    }
    
    private String extractExtension(String filePath) {
        int lastDot = filePath.lastIndexOf('.');
        if (lastDot > 0 && lastDot < filePath.length() - 1) {
            return filePath.substring(lastDot);
        }
        return "";
    }
}
```

### 4.2 MediaService 扩展

在 `MediaService` 中添加获取下载链接和预览链接的方法：

```java
@Service
public class MediaService {
    
    private final MediaFileRepository mediaFileRepository;
    private final SeafileService seafileService;
    
    /**
     * 获取文件下载链接
     * 
     * @param fileId 文件业务ID
     * @return 下载链接
     */
    public String getFileDownloadLink(String fileId) {
        MediaFile file = mediaFileRepository.findByFileId(fileId)
                .orElseThrow(() -> new RuntimeException("文件不存在"));
        
        // 如果是 Seafile 文件，实时获取下载链接
        if (file.getAccessMethod() == AccessMethod.SEAFILE && 
            file.getSeafilePath() != null) {
            return seafileService.getDownloadLink(file.getSeafilePath());
        }
        
        // 否则返回已存储的下载链接
        return file.getDownloadUrl();
    }
    
    /**
     * 获取文件预览链接
     * 
     * @param fileId 文件业务ID
     * @return 预览链接
     */
    public String getFilePreviewUrl(String fileId) {
        MediaFile file = mediaFileRepository.findByFileId(fileId)
                .orElseThrow(() -> new RuntimeException("文件不存在"));
        
        // 如果是 Seafile 文件，生成预览链接
        if (file.getAccessMethod() == AccessMethod.SEAFILE && 
            file.getSeafilePath() != null) {
            return seafileService.generatePreviewUrl(file.getSeafilePath());
        }
        
        // 否则返回已存储的预览链接
        return file.getPreviewUrl();
    }
    
    /**
     * 判断文件是否支持在线预览
     */
    public boolean isFileViewable(String fileId) {
        MediaFile file = mediaFileRepository.findByFileId(fileId)
                .orElseThrow(() -> new RuntimeException("文件不存在"));
        
        if (file.getIsViewable() != null) {
            return file.getIsViewable();
        }
        
        // 如果是 Seafile 文件，实时判断
        if (file.getAccessMethod() == AccessMethod.SEAFILE && 
            file.getSeafilePath() != null) {
            return seafileService.isViewableFile(file.getSeafilePath());
        }
        
        return false;
    }
    
    /**
     * 智能处理文件：返回预览链接或下载链接
     * 
     * @param fileId 文件业务ID
     * @return 访问信息（包含预览链接和下载链接）
     */
    public FileAccessInfo getFileAccessInfo(String fileId) {
        MediaFile file = mediaFileRepository.findByFileId(fileId)
                .orElseThrow(() -> new RuntimeException("文件不存在"));
        
        boolean viewable = isFileViewable(fileId);
        String previewUrl = viewable ? getFilePreviewUrl(fileId) : null;
        String downloadUrl = getFileDownloadLink(fileId);
        
        return FileAccessInfo.builder()
                .fileId(fileId)
                .title(file.getTitle())
                .isViewable(viewable)
                .previewUrl(previewUrl)
                .downloadUrl(downloadUrl)
                .build();
    }
}
```

### 5.4 前端传参说明：如何定位文件

**用于定位文件的字段：`fileId`（接口中体现为 `id`）**

| 场景 | 前端传入方式 | 说明 |
|------|--------------|------|
| 获取详情 | `GET /media/files/{id}` | 路径参数 `id` = 文件的 `fileId` |
| 获取访问信息 | `GET /media/files/{id}/access` | 同上 |
| 下载 | `GET /media/files/{id}/download` | 同上 |
| 预览 | `GET /media/files/{id}/preview` | 同上 |

**前端如何获得 `id`：**

1. **列表接口**：`GET /media/files?category=xxx&query=xxx` 返回的每项均包含 `id`（即 `fileId`），前端点击某条记录后，用该 `id` 请求详情、访问、下载或预览。
2. **详情/访问接口响应**：`data.id` 或 `data.fileId` 亦可作为后续请求的 `id`。

**示例：**

```
1. 请求列表 → GET /media/files?category=sales
   → 返回 data: [{ "id": "KB001", "title": "市场竞争分析报告", ... }, ...]

2. 用户点击「市场竞争分析报告」→ 用 id "KB001" 请求详情
   → GET /media/files/KB001

3. 用户点击「预览」→ 用同一 id 请求预览
   → GET /media/files/KB001/preview  （后端 302 到 Seafile 预览页）
```

**约定：**

- `id` / `fileId` 全局唯一，格式如 `KB` + 6 位（如 `KB001`、`KB12AB`），**不要**与数据库自增主键混淆。
- 仅支持通过 `id`（fileId）定位；**不**通过 `title`、`seafilePath` 等作为路径参数。若需按标题搜索，使用列表接口的 `query` 参数。

## 八、错误处理

### 8.1 常见错误场景

| 错误场景 | 处理方式 |
|----------|----------|
| Seafile API 调用失败 | 返回错误信息，记录日志 |
| 文件不存在 | 返回 404 错误 |
| 文件不支持预览 | 返回 400 错误，提示不支持预览 |
| Token 过期 | 返回 401 错误，提示重新配置 |
| 网络超时 | 返回 500 错误，提示稍后重试 |

### 8.2 异常处理示例

```java
@ControllerAdvice
public class MediaExceptionHandler {
    
    @ExceptionHandler(SeafileApiException.class)
    public ResponseEntity<ApiResponse<Void>> handleSeafileApiException(
            SeafileApiException e) {
        return ResponseEntity.ok(ApiResponse.error(502, 
            "文件服务暂不可用: " + e.getMessage()));
    }
    
    @ExceptionHandler(ResourceNotFoundException.class)
    public ResponseEntity<ApiResponse<Void>> handleResourceNotFound(
            ResourceNotFoundException e) {
        return ResponseEntity.ok(ApiResponse.error(404, e.getMessage()));
    }
}
```

## 九、安全与权限

### 9.1 Token 安全

- **REPO_TOKEN** 仅在后端使用，不得暴露给前端
- 下载链接、预览链接由后端代为请求后返回，前端不直接调用 Seafile API
- 生产环境通过环境变量注入 Token，不写入配置文件

### 9.2 访问控制

- 媒体文件列表、详情、访问信息接口可根据业务需求增加登录校验（`@PreAuthorize` 或拦截器）
- 下载/预览重定向接口可考虑增加**临时凭证**（如短期 JWT），避免链接被滥用
