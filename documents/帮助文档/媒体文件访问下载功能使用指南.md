# 媒体文件访问下载功能使用指南

## 文档说明

本文档详细说明如何使用媒体文件访问与下载功能，包括 API 接口使用、配置说明、常见问题等。

---

## 一、功能概述

### 1.1 功能说明

媒体文件访问下载功能支持：

- **文件列表查询**：按分类、关键词搜索媒体文件
- **文件详情查看**：获取文件的详细信息
- **文件访问信息**：获取文件的预览链接和下载链接
- **文件预览**：支持在线预览（PDF、图片、视频等）
- **文件下载**：支持文件下载功能

### 1.2 支持的访问方式

- **Seafile**：通过 NJU Box (Seafile) 访问文件
- **本地文件**：存储在服务器本地的文件
- **对象存储**：存储在 OSS 等云存储服务的文件

### 1.3 支持的文件类型

| 文件类型 | 扩展名 | 是否可预览 |
|----------|--------|------------|
| PDF | `.pdf` | ✅ 是 |
| 图片 | `.jpg`, `.jpeg`, `.png`, `.gif`, `.bmp`, `.webp`, `.svg` | ✅ 是 |
| 视频 | `.mp4`, `.mov`, `.avi`, `.mkv`, `.flv`, `.wmv`, `.webm`, `.ts` | ✅ 是 |
| 音频 | `.mp3`, `.wav`, `.flac`, `.aac`, `.ogg`, `.m4a` | ✅ 是 |
| Excel | `.xlsx`, `.xls` | ❌ 否 |
| PPT | `.pptx`, `.ppt` | ❌ 否 |

## 二、API 接口说明

### 2.1 基础路径

所有媒体文件相关接口的基础路径为：`/api/cleaner-support/v2/media`

### 2.2 接口列表

#### 2.2.1 获取文件列表

**接口路径**：`GET /api/cleaner-support/v2/media/files`

**请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| category | String | 否 | 文件分类（sales, company, product, maintenance, training） |
| query | String | 否 | 搜索关键词（匹配标题） |

**请求示例**：

```bash
# 获取所有文件
GET /api/cleaner-support/v2/media/files

# 按分类查询
GET /api/cleaner-support/v2/media/files?category=sales

# 关键词搜索
GET /api/cleaner-support/v2/media/files?query=传感器

# 组合查询
GET /api/cleaner-support/v2/media/files?category=maintenance&query=视频
```

**响应示例**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": [
    {
      "id": "KB123456",
      "title": "市场竞争分析报告",
      "type": "PDF",
      "coverUrl": "https://example.com/cover.jpg"
    },
    {
      "id": "KB789ABC",
      "title": "维护视频教程",
      "type": "Video",
      "coverUrl": "https://example.com/video-cover.jpg"
    }
  ]
}
```

#### 2.2.2 获取文件详情

**接口路径**：`GET /api/cleaner-support/v2/media/files/{id}`

**路径参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 文件业务ID（fileId），如 "KB123456" |

**请求示例**：

```bash
GET /api/cleaner-support/v2/media/files/KB123456
```

**响应示例**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "id": "KB123456",
    "mediaUrl": "https://example.com/file.pdf",
    "previewUrl": "https://box.nju.edu.cn/lib/.../file/...",
    "downloadUrl": "https://box.nju.edu.cn/files/...",
    "isViewable": true
  }
}
```

#### 2.2.3 获取文件访问信息

**接口路径**：`GET /api/cleaner-support/v2/media/files/{id}/access`

**路径参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 文件业务ID |

**请求示例**：

```bash
GET /api/cleaner-support/v2/media/files/KB123456/access
```

**响应示例**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "fileId": "KB123456",
    "title": "市场竞争分析报告",
    "isViewable": true,
    "previewUrl": "https://box.nju.edu.cn/lib/9136e9de-f49e-45f6-9aff-9e3abe07a257/file/%2F00-Sales%20Kit%2F00-Competitors%20in%20the%20market.pdf",
    "downloadUrl": "https://box.nju.edu.cn/files/abc123def456..."
  }
}
```

#### 2.2.4 预览文件

**接口路径**：`GET /api/cleaner-support/v2/media/files/{id}/preview`

**路径参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 文件业务ID |

**说明**：
- 如果文件支持预览，接口会返回 HTTP 302 重定向到预览链接
- 如果文件不支持预览，返回错误信息

**请求示例**：

```bash
GET /api/cleaner-support/v2/media/files/KB123456/preview
```

**响应示例**（支持预览）：

```
HTTP/1.1 302 Found
Location: https://box.nju.edu.cn/lib/.../file/...
```

**响应示例**（不支持预览）：

```json
{
  "code": 400,
  "message": "该文件类型不支持在线预览"
}
```

#### 2.2.5 下载文件

**接口路径**：`GET /api/cleaner-support/v2/media/files/{id}/download`

**路径参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 文件业务ID |

**说明**：
- 接口会返回 HTTP 302 重定向到下载链接
- 对于 Seafile 文件，下载链接是临时链接，有时效性

**请求示例**：

```bash
GET /api/cleaner-support/v2/media/files/KB123456/download
```

**响应示例**：

```
HTTP/1.1 302 Found
Location: https://box.nju.edu.cn/files/abc123def456...
```

## 三、配置说明

### 3.1 Seafile 配置

在 `application.yml` 中配置 Seafile 相关参数：

```yaml
seafile:
  server-url: ${SEAFILE_SERVER_URL:https://box.nju.edu.cn}
  repo-token: ${SEAFILE_REPO_TOKEN:your-token-here}
  repo-id: ${SEAFILE_REPO_ID:your-repo-id-here}
  repo-password: ${SEAFILE_REPO_PASSWORD:}
```

### 3.2 环境变量配置

**生产环境建议使用环境变量**，避免敏感信息硬编码：

```bash
# Linux/Mac
export SEAFILE_SERVER_URL=https://box.nju.edu.cn
export SEAFILE_REPO_TOKEN=your-token-here
export SEAFILE_REPO_ID=your-repo-id-here
export SEAFILE_REPO_PASSWORD=your-password-if-needed

# Windows
set SEAFILE_SERVER_URL=https://box.nju.edu.cn
set SEAFILE_REPO_TOKEN=your-token-here
set SEAFILE_REPO_ID=your-repo-id-here
set SEAFILE_REPO_PASSWORD=your-password-if-needed
```

### 3.3 配置参数说明

| 参数名 | 说明 | 必填 | 默认值 |
|--------|------|------|--------|
| `server-url` | Seafile 服务器地址 | 是 | `https://box.nju.edu.cn` |
| `repo-token` | Repository Token（资料库令牌） | 是 | 无 |
| `repo-id` | 资料库ID（UUID格式） | 是 | 无 |
| `repo-password` | 资料库密码（如果加密） | 否 | 空 |

## 四、前端使用示例

### 4.1 获取文件列表

```typescript
// 使用 fetch
const response = await fetch('/api/cleaner-support/v2/media/files?category=sales');
const result = await response.json();

if (result.code === 200) {
  const files = result.data;
  // 处理文件列表
  files.forEach(file => {
    console.log(file.id, file.title);
  });
}
```

```typescript
// 使用 axios
import axios from 'axios';

const response = await axios.get('/api/cleaner-support/v2/media/files', {
  params: {
    category: 'sales',
    query: '传感器'
  }
});

if (response.data.code === 200) {
  const files = response.data.data;
  // 处理文件列表
}
```

### 4.2 获取文件详情

```typescript
const fileId = 'KB123456';
const response = await fetch(`/api/cleaner-support/v2/media/files/${fileId}`);
const result = await response.json();

if (result.code === 200) {
  const fileDetail = result.data;
  console.log('预览链接:', fileDetail.previewUrl);
  console.log('下载链接:', fileDetail.downloadUrl);
  console.log('是否可预览:', fileDetail.isViewable);
}
```

### 4.3 获取文件访问信息

```typescript
const fileId = 'KB123456';
const response = await fetch(`/api/cleaner-support/v2/media/files/${fileId}/access`);
const result = await response.json();

if (result.code === 200) {
  const accessInfo = result.data;
  
  // 如果可预览，显示预览按钮
  if (accessInfo.isViewable) {
    // 预览链接：accessInfo.previewUrl
  }
  
  // 下载链接：accessInfo.downloadUrl
}
```

### 4.4 预览文件

```typescript
const fileId = 'KB123456';

// 方式1：直接重定向（推荐）
window.location.href = `/api/cleaner-support/v2/media/files/${fileId}/preview`;

// 方式2：在新窗口打开
window.open(`/api/cleaner-support/v2/media/files/${fileId}/preview`, '_blank');

// 方式3：使用 iframe 嵌入
const iframe = document.createElement('iframe');
iframe.src = `/api/cleaner-support/v2/media/files/${fileId}/preview`;
document.body.appendChild(iframe);
```

### 4.5 下载文件

```typescript
const fileId = 'KB123456';

// 方式1：直接重定向（推荐）
window.location.href = `/api/cleaner-support/v2/media/files/${fileId}/download`;

// 方式2：使用 a 标签
const link = document.createElement('a');
link.href = `/api/cleaner-support/v2/media/files/${fileId}/download`;
link.download = 'filename.pdf'; // 可选：指定下载文件名
link.click();

// 方式3：先获取下载链接，再下载
const response = await fetch(`/api/cleaner-support/v2/media/files/${fileId}/access`);
const result = await response.json();
if (result.code === 200) {
  window.location.href = result.data.downloadUrl;
}
```

### 4.6 完整示例：文件列表页面

```typescript
import React, { useState, useEffect } from 'react';
import axios from 'axios';

function MediaFileList() {
  const [files, setFiles] = useState([]);
  const [category, setCategory] = useState('');
  const [query, setQuery] = useState('');

  useEffect(() => {
    loadFiles();
  }, [category, query]);

  const loadFiles = async () => {
    try {
      const response = await axios.get('/api/cleaner-support/v2/media/files', {
        params: { category, query }
      });
      if (response.data.code === 200) {
        setFiles(response.data.data);
      }
    } catch (error) {
      console.error('加载文件列表失败:', error);
    }
  };

  const handlePreview = (fileId: string) => {
    window.open(`/api/cleaner-support/v2/media/files/${fileId}/preview`, '_blank');
  };

  const handleDownload = (fileId: string) => {
    window.location.href = `/api/cleaner-support/v2/media/files/${fileId}/download`;
  };

  return (
    <div>
      <div>
        <select value={category} onChange={(e) => setCategory(e.target.value)}>
          <option value="">全部</option>
          <option value="sales">销售资料</option>
          <option value="maintenance">维护资料</option>
        </select>
        <input
          type="text"
          value={query}
          onChange={(e) => setQuery(e.target.value)}
          placeholder="搜索关键词"
        />
      </div>
      
      <ul>
        {files.map(file => (
          <li key={file.id}>
            <h3>{file.title}</h3>
            <p>类型: {file.type}</p>
            <button onClick={() => handlePreview(file.id)}>预览</button>
            <button onClick={() => handleDownload(file.id)}>下载</button>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

## 五、后端开发说明

### 5.1 实体类字段说明

`MediaFile` 实体类包含以下关键字段：

- **fileId**：业务ID，用于 API 接口标识
- **title**：文件标题
- **type**：文件类型（FileType 枚举）
- **category**：文件分类
- **accessMethod**：访问方式（AccessMethod 枚举）
- **seafilePath**：Seafile 文件路径（当 accessMethod = SEAFILE 时使用）
- **repoId**：Seafile 资料库ID
- **isViewable**：是否支持在线预览
- **previewUrl**：预览链接
- **downloadUrl**：下载链接

### 5.2 服务类说明

#### SeafileService

负责与 Seafile API 交互：

- `getDownloadLink(String filePath)`：获取文件下载链接
- `generatePreviewUrl(String filePath)`：生成预览链接
- `isViewableFile(String filePath)`：判断文件是否可预览

#### MediaService

负责媒体文件业务逻辑：

- `searchFiles(String category, String query)`：搜索文件列表
- `getFileDetail(String fileId)`：获取文件详情
- `getFileAccessInfo(String fileId)`：获取文件访问信息
- `getFileDownloadLink(String fileId)`：获取下载链接
- `getFilePreviewUrl(String fileId)`：获取预览链接
- `isFileViewable(String fileId)`：判断是否可预览

### 5.3 创建 MediaFile 实体示例

```java
// 创建 Seafile 文件
MediaFile file = MediaFile.builder()
    .fileId("KB123456")
    .title("市场竞争分析报告")
    .type(MediaFile.FileType.PDF)
    .category("sales")
    .repoId("9136e9de-f49e-45f6-9aff-9e3abe07a257")
    .seafilePath("/00-Sales Kit/00-Competitors in the market.pdf")
    .isViewable(true)
    .accessMethod(MediaFile.AccessMethod.SEAFILE)
    .build();

mediaFileRepository.save(file);
```

## 六、常见问题

### 6.1 如何获取文件的 fileId？

**答**：通过文件列表接口获取。列表接口返回的每项都包含 `id` 字段，这就是 `fileId`。

```typescript
// 获取列表
const response = await fetch('/api/cleaner-support/v2/media/files');
const result = await response.json();
const files = result.data;

// 使用第一个文件的 id
const fileId = files[0].id; // 例如 "KB123456"
```

### 6.2 文件预览失败怎么办？

**可能原因**：
1. 文件类型不支持预览（如 Excel、PPT）
2. Seafile 服务器连接失败
3. 文件路径不存在

**解决方法**：
1. 检查 `isViewable` 字段，确认文件是否支持预览
2. 检查 Seafile 配置是否正确
3. 查看后端日志，确认具体错误信息

### 6.3 下载链接失效怎么办？

**说明**：Seafile 的下载链接是临时链接，有时效性。如果链接失效，需要重新调用接口获取新的下载链接。

**解决方法**：每次下载前都调用 `/files/{id}/download` 接口，让后端实时获取新的下载链接。

### 6.4 如何判断文件是否支持预览？

**方法1**：通过访问信息接口

```typescript
const response = await fetch(`/api/cleaner-support/v2/media/files/${fileId}/access`);
const result = await response.json();
const isViewable = result.data.isViewable;
```

**方法2**：通过文件详情接口

```typescript
const response = await fetch(`/api/cleaner-support/v2/media/files/${fileId}`);
const result = await response.json();
const isViewable = result.data.isViewable;
```

### 6.5 如何区分不同的访问方式？

通过文件详情接口的 `accessMethod` 字段（如果接口返回）或通过文件路径判断：

- 如果 `seafilePath` 不为空，说明是 Seafile 文件
- 如果 `filePath` 不为空，说明是本地文件
- 如果 `mediaUrl` 不为空且不是 Seafile 链接，说明是 OSS 文件

## 七、错误码说明

| 错误码 | 说明 | 解决方法 |
|--------|------|----------|
| 200 | 成功 | - |
| 400 | 请求参数错误 | 检查请求参数是否正确 |
| 404 | 文件不存在 | 确认 fileId 是否正确 |
| 500 | 服务器错误 | 查看后端日志，检查 Seafile 配置 |
| 502 | Seafile 服务不可用 | 检查 Seafile 服务器连接和配置 |

## 八、最佳实践

### 8.1 前端建议

1. **列表查询**：使用分页，避免一次性加载大量数据
2. **预览文件**：在新窗口打开，避免影响当前页面
3. **下载文件**：显示下载进度（如果可能）
4. **错误处理**：友好提示用户错误信息
5. **缓存策略**：文件列表可以适当缓存，但访问链接不要缓存（因为有时效性）

### 8.2 后端建议

1. **配置安全**：生产环境使用环境变量，不要硬编码敏感信息
2. **错误处理**：记录详细日志，便于排查问题
3. **性能优化**：为常用查询字段添加索引
4. **链接缓存**：可以考虑缓存下载链接（但要注意时效性）
