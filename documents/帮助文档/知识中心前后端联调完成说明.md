## 二、接口对接说明

### 2.1 文件列表接口

**前端调用**：
```typescript
const files = await getMediaFiles({
  category: 'sales',  // 可选
  query: '传感器'     // 可选
});
```

**后端接口**：`GET /api/cleaner-support/v2/media/files?category=sales&query=传感器`

**响应格式**：
```json
{
  "code": 200,
  "data": [
    {
      "id": "KB123456",
      "title": "市场竞争分析报告",
      "type": "PDF",
      "coverUrl": "https://example.com/cover.jpg"
    }
  ]
}
```

### 2.2 文件详情接口

**前端调用**：
```typescript
const detail = await getMediaFileDetail('KB123456');
```

**后端接口**：`GET /api/cleaner-support/v2/media/files/KB123456`

**响应格式**：
```json
{
  "code": 200,
  "data": {
    "id": "KB123456",
    "mediaUrl": "https://example.com/file.pdf",
    "previewUrl": "https://box.nju.edu.cn/lib/.../file/...",
    "downloadUrl": "https://box.nju.edu.cn/files/...",
    "isViewable": true
  }
}
```

### 2.3 文件访问信息接口

**前端调用**：
```typescript
const accessInfo = await getFileAccessInfo('KB123456');
```

**后端接口**：`GET /api/cleaner-support/v2/media/files/KB123456/access`

**响应格式**：
```json
{
  "code": 200,
  "data": {
    "fileId": "KB123456",
    "title": "市场竞争分析报告",
    "isViewable": true,
    "previewUrl": "https://box.nju.edu.cn/lib/.../file/...",
    "downloadUrl": "https://box.nju.edu.cn/files/..."
  }
}
```

### 2.4 预览和下载接口

**前端调用**：
```typescript
previewFile('KB123456');   // 在新窗口打开预览
downloadFile('KB123456');  // 下载文件
```

**后端接口**：
- `GET /api/cleaner-support/v2/media/files/KB123456/preview` → HTTP 302 重定向
- `GET /api/cleaner-support/v2/media/files/KB123456/download` → HTTP 302 重定向

## 三、使用流程

### 3.1 用户操作流程

1. **打开知识中心**：用户点击底部导航栏的"知识库"图标
2. **浏览文件列表**：系统自动加载所有文件，用户可以看到文件列表
3. **筛选文件**：
   - 点击分类标签（全部、销售资料、维护保养等）进行筛选
   - 在搜索框输入关键词进行搜索
4. **查看文件详情**：点击文件卡片，弹出详情对话框
5. **预览/下载文件**：
   - 如果文件支持预览，点击"预览"按钮在新窗口打开
   - 点击"下载"按钮下载文件

### 3.2 数据流程

```
用户操作
  ↓
前端 KnowledgePage 组件
  ↓
调用 getMediaFiles() API
  ↓
后端 MediaController.listFiles()
  ↓
MediaService.searchFiles()
  ↓
MediaFileRepository 查询数据库
  ↓
返回文件列表数据
  ↓
前端展示文件列表
```

**文件详情流程**：
```
用户点击文件
  ↓
调用 getFileAccessInfo() API
  ↓
后端 MediaController.getFileAccess()
  ↓
MediaService.getFileAccessInfo()
  ↓
SeafileService 获取访问链接（如果是 Seafile 文件）
  ↓
返回访问信息（预览链接、下载链接）
  ↓
前端显示预览/下载按钮
```

- `PDF` → 文件图标

- `PPT` → 演示文稿图标

## 六、错误处理

### 6.1 前端错误处理

- **加载失败**：显示错误提示和重试按钮
- **文件不存在**：显示"未找到相关内容"
- **获取访问信息失败**：在详情对话框中显示"无法获取文件访问信息"

### 6.2 后端错误处理

- **文件不存在**：返回 404 错误
- **Seafile API 调用失败**：返回 500 错误，记录日志
- **文件不支持预览**：返回 400 错误

## 七、配置要求

### 7.1 后端配置

在 `application.yml` 中配置 Seafile 参数：

```yaml
seafile:
  server-url: ${SEAFILE_SERVER_URL:https://box.nju.edu.cn}
  repo-token: ${SEAFILE_REPO_TOKEN:your-token-here}
  repo-id: ${SEAFILE_REPO_ID:your-repo-id-here}
  repo-password: ${SEAFILE_REPO_PASSWORD:}
```

### 7.2 前端配置

在 `.env` 或环境变量中配置 API 基础路径：

```bash
VITE_API_BASE_URL=http://localhost:8080/api/cleaner-support/v2
```

## 九、已知问题和注意事项

### 9.1 注意事项

1. **Seafile 下载链接时效性**：Seafile 的下载链接是临时链接，有时效性，需要实时获取
2. **预览链接**：预览链接需要用户已登录 Seafile 才能正常预览
3. **分类映射**：确保前端分类和后端分类的映射关系正确
4. **错误处理**：前端需要处理各种错误情况，提供友好的用户提示

### 9.2 待优化项

1. **分页功能**：当前文件列表没有分页，如果文件数量很多，建议添加分页
2. **缓存策略**：可以考虑缓存文件列表，但访问链接不要缓存
3. **加载优化**：可以考虑使用虚拟滚动优化长列表性能
4. **预览优化**：可以考虑使用 iframe 嵌入预览，而不是新窗口打开

## 十一、使用示例

```typescript

downloadFile('KB123456');
```

```java
// 在 Service 中使用

private MediaService mediaService;

// 获取文件访问信息
FileAccessInfo accessInfo = mediaService.getFileAccessInfo("KB123456");

String downloadUrl = mediaService.getFileDownloadLink("KB123456");
```
