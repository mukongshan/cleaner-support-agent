# 媒体文件实体类重构说明

## 重构日期
2024-01-23

## 重构原因

原设计中，`MediaFile` 实体类直接存储了以下字段：
- `previewUrl`：预览链接
- `downloadUrl`：下载链接
- `mediaUrl`：媒体文件URL
- `repoId`：Seafile 资料库ID

**问题**：
1. **临时链接有时效性**：`previewUrl` 和 `downloadUrl` 是临时链接，存储后很快就会失效
2. **数据冗余**：`repoId` 是配置信息，不需要每个文件都存储
3. **URL 可生成**：`mediaUrl` 可以通过文件路径生成，不需要存储
4. **违反设计原则**：应该存储路径信息，通过服务生成访问链接

## 重构内容

### 1. 移除的字段

| 字段名 | 原因 | 替代方案 |
|--------|------|----------|
| `previewUrl` | 临时链接，有时效性 | 通过 `SeafileService.generatePreviewUrl()` 实时生成 |
| `downloadUrl` | 临时链接，有时效性 | 通过 `SeafileService.getDownloadLink()` 实时生成 |
| `mediaUrl` | 可通过路径生成 | 通过 `MediaService.generateMediaUrl()` 实时生成 |
| `repoId` | 配置信息，不需要每个文件存储 | 从 `application.yml` 配置中读取 |

### 2. 新增的字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `storageKey` | `String(500)` | 对象存储的 key（当 `accessMethod = OSS` 时使用） |

### 3. 保留的字段

| 字段名 | 说明 |
|--------|------|
| `seafilePath` | Seafile 文件路径（当 `accessMethod = SEAFILE` 时使用） |
| `filePath` | 本地文件路径（当 `accessMethod = LOCAL` 时使用） |
| `coverUrl` | 封面图URL（可选，因为封面图通常是静态资源） |
| `isViewable` | 是否可预览（可根据文件类型自动判断） |

## 服务层改动

### MediaService 新增方法

1. **`generateMediaUrl(MediaFile file)`**
   - 根据访问方式生成媒体URL
   - Seafile：返回预览链接
   - LOCAL：生成本地文件访问URL
   - OSS：生成OSS访问URL

2. **`generateLocalFileUrl(String filePath)`**
   - 生成本地文件访问URL
   - 将本地路径转换为HTTP访问路径

3. **`generateOssFileUrl(String storageKey)`**
   - 生成OSS文件访问URL（待实现）

4. **`getFilePath(MediaFile file)`**
   - 根据访问方式获取实际文件路径

5. **`isViewableByExtension(String path)`**
   - 根据文件扩展名判断是否可预览

### 修改的方法

1. **`getFileDownloadLink(String fileId)`**
   - 根据访问方式实时生成下载链接
   - Seafile：调用 `SeafileService.getDownloadLink()`
   - LOCAL：调用 `generateLocalFileUrl()`
   - OSS：调用 `generateOssFileUrl()`

2. **`getFilePreviewUrl(String fileId)`**
   - 根据访问方式实时生成预览链接
   - Seafile：调用 `SeafileService.generatePreviewUrl()`
   - LOCAL：调用 `generateLocalFileUrl()`
   - OSS：调用 `generateOssFileUrl()`

3. **`isFileViewable(String fileId)`**
   - 根据文件路径判断是否可预览
   - 支持 Seafile、本地文件、OSS 文件

## 数据库迁移

### 迁移脚本

已创建迁移脚本：`src/backend/cleaner-support-agent-backend/src/main/resources/db/migration_remove_url_fields.sql`

**执行步骤**：

```sql
-- 1. 备份数据（可选）
CREATE TABLE media_files_backup AS SELECT * FROM media_files;

-- 2. 移除不需要的字段
ALTER TABLE media_files 
    DROP COLUMN IF EXISTS preview_url,
    DROP COLUMN IF EXISTS download_url,
    DROP COLUMN IF EXISTS media_url,
    DROP COLUMN IF EXISTS repo_id;

-- 3. 添加新字段（如果不存在）
ALTER TABLE media_files 
    ADD COLUMN IF NOT EXISTS storage_key VARCHAR(500) COMMENT '对象存储的 key（当 access_method = OSS 时使用）';
```

### 注意事项

1. **数据备份**：执行迁移前建议先备份数据
2. **JPA 自动更新**：如果使用 `ddl-auto: update`，JPA 会自动更新表结构
3. **现有数据**：迁移后，现有数据的访问链接需要通过服务重新生成

## 使用示例

### 创建 Seafile 文件

```java
MediaFile file = MediaFile.builder()
    .fileId("KB123456")
    .title("市场竞争分析报告")
    .type(MediaFile.FileType.PDF)
    .category("sales")
    .seafilePath("/00-Sales Kit/00-Competitors in the market.pdf")
    .isViewable(true)
    .accessMethod(MediaFile.AccessMethod.SEAFILE)
    .build();

mediaFileRepository.save(file);

// 获取访问链接（实时生成）
String previewUrl = mediaService.getFilePreviewUrl("KB123456");
String downloadUrl = mediaService.getFileDownloadLink("KB123456");
```

### 创建本地文件

```java
MediaFile file = MediaFile.builder()
    .fileId("KB789ABC")
    .title("维护视频教程")
    .type(MediaFile.FileType.Video)
    .category("maintenance")
    .filePath("./data/uploads/maintenance_video.mp4")
    .coverUrl("/api/media/files/KB789ABC/cover.jpg")
    .isViewable(true)
    .accessMethod(MediaFile.AccessMethod.LOCAL)
    .build();

mediaFileRepository.save(file);

// 获取访问链接（实时生成）
String mediaUrl = mediaService.generateMediaUrl(file);
```

## 优势

1. **数据一致性**：不存储临时链接，避免链接失效问题
2. **减少存储**：移除冗余字段，减少数据库存储空间
3. **灵活性**：链接生成逻辑集中管理，易于修改和维护
4. **安全性**：临时链接实时生成，提高安全性
5. **配置集中**：`repoId` 等配置信息集中管理

## 注意事项

1. **性能考虑**：每次获取链接都需要调用服务，可能影响性能
   - **解决方案**：可以考虑添加缓存（但要注意时效性）

2. **错误处理**：服务调用失败时需要妥善处理
   - **解决方案**：添加异常处理和重试机制

3. **OSS 服务**：`generateOssFileUrl()` 方法需要实现
   - **待实现**：根据实际使用的 OSS 服务实现
