# 开发环境上手指南（后端 · cleaner-support-agent-backend）

## 1. 前置条件

- 操作系统：Windows 10/11（其他系统思路类似）
- 已安装：
  - JDK 21+（推荐 21 或 22）
  - Git
  - MySQL 8+（本地实例，端口默认 3306）
  - PowerShell（默认自带）

> 如果你只做后端开发，前端（Flutter）环境可以后面再装。

---

## 2. 拉取代码

```bash
git clone <仓库地址>
cd cleaner-support-agent
```

后端代码路径：

```text
src/backend/cleaner-support-agent-backend
```

后面的命令都在这个目录下执行。

---

## 3. 配置环境变量（set-env.ps1）

在 `src/backend/cleaner-support-agent-backend` 目录下有一个脚本：

```text
set-env.ps1
```

它会为当前 PowerShell 会话设置后端运行依赖的环境变量：

```powershell
# 数据库
$env:SPRING_DATASOURCE_URL="jdbc:mysql://localhost:3306/cleaner_support?useUnicode=true&characterEncoding=utf8&useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=Asia/Shanghai"
$env:SPRING_DATASOURCE_USERNAME="kongshan"
$env:SPRING_DATASOURCE_PASSWORD="zmt20041204"

# AI服务
$env:APP_AI_BASE_URL="https://api.dify.ai/v1"
$env:APP_AI_API_KEY="your_api_key"

# 前端地址
$env:APP_CORS_ALLOWED_ORIGINS="http://localhost:5173"

# 启动端口
$env:SERVER_PORT="8080"
```

### 3.1 在 PowerShell 中执行

在项目根目录打开 PowerShell，进入后端目录：

```powershell
cd src/backend/cleaner-support-agent-backend
```

第一次在本机执行脚本，可能需要允许执行本地脚本（如已配置可跳过）：

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

然后执行：

```powershell
.\set-env.ps1
```

看到绿色提示 `Environment variables set successfully!` 即表示当前 PowerShell 会话中的环境变量已生效。

> 注意：这些变量**只在当前 PowerShell 窗口里有效**，关闭窗口后需要重新执行一次。

### 3.2 如需修改数据库账号

如果你本机的 MySQL 账号不是 `kongshan/zmt20041204`，可以：

- 直接修改 `set-env.ps1` 中的：
  - `$env:SPRING_DATASOURCE_USERNAME`
  - `$env:SPRING_DATASOURCE_PASSWORD`
- 或在执行脚本前手动设置：

```powershell
$env:SPRING_DATASOURCE_USERNAME="your_username"
$env:SPRING_DATASOURCE_PASSWORD="your_password"
.\set-env.ps1
```

## 4. 初始化 MySQL 数据库

后端默认连接的数据库配置（见 `application.yml` 与 `set-env.ps1`）：

- 地址：`jdbc:mysql://localhost:3306/cleaner_support?...`
- 数据库名：`cleaner_support`
- 字符集：`utf8mb4`

实体（Entity）表结构由 JPA 自动创建，你只需要先**创建数据库本身**。

### 4.1 确保 MySQL 已启动

- Windows 服务方式：

```powershell
net start MySQL80   # 或你的 MySQL 服务名
```

- 或通过任务栏 / MySQL Workbench 启动。

### 4.2 使用命令行创建数据库

在 PowerShell 或 CMD 中执行：

```powershell
mysql -u kongshan -p
# 输入密码：zmt20041204（或你在 set-env.ps1 中配置的密码）
```

登录后执行：

```sql
CREATE DATABASE IF NOT EXISTS cleaner_support
    CHARACTER SET utf8mb4
    COLLATE utf8mb4_unicode_ci;
```

验证：

```sql
SHOW DATABASES;
USE cleaner_support;
SHOW TABLES;  -- 此时通常是空的，启动应用后 JPA 会自动建表
```

### 4.3 使用 Workbench / GUI 创建（可选）

1. 打开 MySQL Workbench
2. 连接到本地 MySQL
3. 右键左侧空白处 → `Create Schema`
4. 填写：
   - Schema 名：`cleaner_support`
   - Charset：`utf8mb4`
   - Collation：`utf8mb4_unicode_ci`
5. Apply 保存

## 5. 启动后端服务

确保已完成：

1. 在 **当前 PowerShell 会话** 中执行过 `.\set-env.ps1`
2. MySQL 中已存在数据库 `cleaner_support`

然后在 `src/backend/cleaner-support-agent-backend` 目录执行：

```powershell
.\mvnw.cmd spring-boot:run
```

首次运行会下载依赖，时间稍长；看到类似输出即表示启动成功：

```text
:: Spring Boot ::  (v3.2.0)
Started CleanerSupportAgentBackendApplication in X.XXX seconds
```

### 5.1 验证服务是否启动成功

- 浏览器访问：

```text
http://localhost:8080/actuator/health
```

如果看到类似：

```json
{"status":"UP"}
```

说明应用健康。

> 如果还没加 Actuator，可以先用 Postman / curl 访问任意已实现的接口，例如登录接口。

## 6. 快速验证几个核心接口

以下示例使用 PowerShell 下的 `curl`（别名为 `Invoke-WebRequest`，可以换成 Postman）。

### 6.1 登录

```powershell
curl -Method POST `
  -Uri "http://localhost:8080/api/cleaner-support/v2/users/login" `
  -ContentType "application/json" `
  -Body '{"username":"13800138000","password":"pwd","loginType":"password"}'
```

返回示例（部分）：

```json
{
  "code": 200,
  "data": {
    "token": "mock_token_...",
    "userId": "Uxxxxxx",
    "nickname": "用户8000"
  }
}
```

记录下 `token`，后续接口需要带上：

```text
Authorization: Bearer mock_token_...
```

### 6.2 访问媒体列表（无需登录）

```powershell
curl "http://localhost:8080/api/cleaner-support/v2/media/files"
```

期待返回 `code = 200` 和一个数组（刚开始可能是空数组）。

### 6.3 上传图片（需要登录）

```powershell
$token = "Bearer mock_token_..."  # 替换成上一步获取的 token

curl -Method POST `
  -Uri "http://localhost:8080/api/cleaner-support/v2/media/upload" `
  -Headers @{ Authorization = $token } `
  -ContentType "multipart/form-data" `
  -InFile "D:\path\to\your\image.jpg"
```

（也可以用 Postman 选择 form-data → file 类型来上传）

返回示例：

```json
{
  "code": 200,
  "data": {
    "url": "/api/cleaner-support/v2/media/files/xxxxxx",
    "fileType": "image"
  }
}
```

## 7. 常见问题（FAQ）

### 7.1 启动时报 “Unknown database 'cleaner_support'”

说明数据库还没创建，回到 **第 4 步** 创建数据库即可。

### 7.2 启动时报 “Access denied for user …”

- 检查 `set-env.ps1` 中的：
  - `$env:SPRING_DATASOURCE_USERNAME`
  - `$env:SPRING_DATASOURCE_PASSWORD`
- 确认该用户在 MySQL 中存在，并且有访问 / 创建表的权限。

### 7.3 启动后访问接口返回 500 `"用户不存在"` 等业务错误

- 说明应用已经连上数据库并跑起来了，只是部分业务逻辑（例如用户资料/会话/工单）还在迭代中。
- 后续开发可以参考 `doc/测试结果与分析.md` 了解哪些模块已完成、哪些还在 TODO。

## 8. 面向新接手同学的小提示

1. **优先保证环境能跑起来**：照本文档做到 “能启动 + 能打到任意一个接口” 即算成功上手。
2. **再关注业务细节**：具体接口的返回内容是否完全符合产品文档，可以在环境 OK 后再逐步校正。
3. 如有任何环境相关问题（启动失败、连不上 MySQL 等），优先排查：
   - 当前 PowerShell 是否执行了 `set-env.ps1`
   - MySQL 是否启动、数据库是否创建
   - 端口 `8080` 是否被占用。
