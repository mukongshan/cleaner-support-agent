# 测试问题修复说明

## 问题描述

运行图片识别功能测试时出现以下错误：
```
Unable to determine Dialect without JDBC metadata 
(please set 'javax.persistence.jdbc.url', 'hibernate.connection.url', or 'hibernate.dialect')
```

## 问题原因

测试环境缺少数据库配置，Spring Boot 无法确定使用哪个数据库方言。

## 修复方案

### 1. 添加 H2 数据库依赖

在 `pom.xml` 中添加了 H2 内存数据库依赖（仅用于测试）：

```xml
<!-- H2 数据库（测试用） -->
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <scope>test</scope>
</dependency>
```

### 2. 激活测试配置

在 `BaseControllerTest.java` 中添加了 `@ActiveProfiles("test")` 注解，使测试使用 `application-test.yml` 配置：

```java
@SpringBootTest
@AutoConfigureMockMvc
@ActiveProfiles("test")
public abstract class BaseControllerTest {
    // ...
}
```

### 3. 测试配置说明

测试配置文件 `src/test/resources/application-test.yml` 已配置：
- 使用 H2 内存数据库（无需外部数据库）
- 自动创建和删除表结构
- 配置了图片识别相关的测试参数

## 运行测试

修复后，可以正常运行测试：

```bash
cd src/backend/cleaner-support-agent-backend
mvn test -Dtest=ImageRecognitionControllerTest
```

或者运行所有测试：

```bash
mvn test
```

## 注意事项

1. **H2 数据库**：测试使用 H2 内存数据库，测试结束后数据会自动清除
2. **测试配置**：所有继承 `BaseControllerTest` 的测试类都会自动使用测试配置
3. **API Key**：测试配置中包含了 Qwen-VL API Key，实际测试时会调用真实 API（如果配置了的话）

## 验证修复

运行测试后，应该看到：
- ✅ 所有测试用例通过
- ✅ 没有数据库连接错误
- ✅ ApplicationContext 成功加载
