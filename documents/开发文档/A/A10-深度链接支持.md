# A10：深度链接支持

## 修改点

> 支持深度链接：直接打开 `/tickets/:id` 等 URL 时正确展示详情，并完成鉴权与 404 处理。

## 实现说明

### 1. 直接打开详情 URL

- **路由**：`/tickets/:id` 对应 **`TicketDetailRoute`**（`src/frontend/src/pages/TicketDetailRoute.tsx`）。  
- **行为**：  
  - 通过 **`useParams().id`** 获取工单 ID。  
  - 调用 **`getTicketDetail(id)`** 拉取详情。  
  - **成功**：映射为页面 `Ticket` 并渲染 **`TicketDetailPage`**。  
  - **接口 404**（或资源不存在）：渲染 **`<NotFoundPage message="工单不存在" />`**（见 A6）。  
- 不依赖「从工单列表点进」的上下文，直接通过 URL 即可打开详情。

### 2. 鉴权

- **`ProtectedRoute`** 包装主布局（含 `/tickets/:id`）。  
- 未登录用户直接访问 **`/tickets/123`** 时：  
  - 先被 **`ProtectedRoute`** 拦截，跳转 **`/login?redirect=/tickets/123`**。  
  - 登录成功后根据 **`redirect`** 跳回 **`/tickets/123`**，再由 **`TicketDetailRoute`** 拉取详情并展示（见 A9）。

### 3. 404 处理

- 工单不存在或接口返回 404 时，在 **`TicketDetailRoute`** 内展示 **「工单不存在」** 的 404 页，不静默跳转首页（见 A6）。

### 4. 验收要点

- 在未登录状态下，于新标签页或地址栏直接打开 **`/tickets/某id`**：先到登录页并带 `redirect`，登录后进入该工单详情页。  
- 直接打开存在的工单 ID：若已登录，则正常展示详情；若未登录，则先登录再回跳并展示。  
- 直接打开不存在的工单 ID：展示「工单不存在」页，不静默跳转首页。
