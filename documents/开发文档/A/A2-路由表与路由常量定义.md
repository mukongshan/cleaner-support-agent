# A2：路由表与路由常量定义

## 修改点

> 定义路由表：`/` 或 `/chat`（问答）、`/knowledge`、`/knowledge/:id`（若产品需要）、`/tickets`、`/tickets/:id`、`/profile`、`/login`、`/register`。

## 实现说明

### 1. 路由常量 `src/frontend/src/constants/routes.ts`

- **路径常量**  
  - `ROUTES.CHAT` = `'/chat'`  
  - `ROUTES.KNOWLEDGE` = `'/knowledge'`  
  - `ROUTES.KNOWLEDGE_DETAIL` = `'/knowledge/:id'`（暂不启用）  
  - `ROUTES.TICKETS` = `'/tickets'`  
  - `ROUTES.TICKET_DETAIL` = `'/tickets/:id'`  
  - `ROUTES.PROFILE` = `'/profile'`  
  - `ROUTES.LOGIN` = `'/login'`  
  - `ROUTES.REGISTER` = `'/register'`

- **工单详情路径生成**  
  - `ticketDetailPath(id: string)`：返回 `/tickets/${encodeURIComponent(id)}`，用于跳转或链接。

- **约定**  
  - 禁止在组件或路由中手写路径字符串，统一从 `constants/routes.ts` 引用。

### 2. 路由表（`src/frontend/src/routes/index.tsx`）

| 路径 | 说明 |
|------|------|
| `/` | 重定向到 `/chat`（index 路由） |
| `/chat` | 问答页 |
| `/knowledge` | 知识库 |
| `/tickets` | 工单列表 |
| `/tickets/:id` | 工单详情（由 `TicketDetailRoute` 拉取数据并处理 404） |
| `/profile` | 个人中心 |
| `/login` | 登录页 |
| `/register` | 注册页 |
| `*` | 404 页（见 A5） |

- `/knowledge/:id` 暂不在路由表中启用，待产品确认后再加。

### 3. 验收要点

- 所有跳转、链接、重定向均使用 `ROUTES.*` 或 `ticketDetailPath(id)`。
- 根路径 `/` 访问时自动跳转到 `/chat`。
