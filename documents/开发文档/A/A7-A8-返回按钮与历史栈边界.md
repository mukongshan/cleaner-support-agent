# A7 / A8：返回按钮与历史栈边界

## 修改点

- **A7**：所有页面内「返回」按钮统一使用 `navigate(-1)`（或等价），与浏览器后退行为一致。  
- **A8**：当历史栈上一页为站外时，对 `navigate(-1)` 做边界处理：不弹窗，改为跳转上一级路径（如 `/tickets/123` → `/tickets`）。

## 实现说明

### 1. 统一返回逻辑：`useBackOrFallback`

- 位置：**`src/frontend/src/hooks/useBackOrFallback.ts`**  
- 签名：`useBackOrFallback(parentPath: string): () => void`  
- 行为：  
  - **`window.history.length > 1`**：调用 **`navigate(-1)`**，与浏览器后退一致。  
  - **`window.history.length <= 1`**（如直接打开链接、从站外进入）：不执行 `navigate(-1)`，改为 **`navigate(parentPath)`**，避免用户离开本站。  
- 约定：采用「不弹窗、改为上一级路径」方案（与 11.2 约定一致）。

### 2. 使用方式

- **工单详情页**（`TicketDetailRoute` 渲染的 `TicketDetailPage`）：  
  - 传入 **`onBack={useBackOrFallback(ROUTES.TICKETS)}`**，即上一级为工单列表。  
- 其他带「返回」的页面（如从列表进入的二级页）：  
  - 同样使用 `useBackOrFallback(上一级路径)` 或直接 `() => navigate(-1)`（若无需考虑站外进入）。

### 3. 验收要点

- 从站内依次进入：首页 → 工单列表 → 工单详情，在详情页点「返回」回到列表，再点「返回」回到上一级，与浏览器后退一致。
- 直接打开或从外链进入工单详情（历史栈仅当前页或上一页为站外），点「返回」时进入工单列表 `/tickets`，而不是离开本站。
