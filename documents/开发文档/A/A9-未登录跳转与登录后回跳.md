# A9：未登录跳转与登录后回跳

## 修改点

> 未登录用户访问需登录的 URL 时，跳转 `/login?redirect=当前完整 URL`；登录成功后根据 `redirect` 参数跳回原页面，无则回默认页。

## 实现说明

### 1. 需登录路径与重定向

- **`src/frontend/src/constants/routes.ts`**  
  - 定义 **`isAuthRequiredPath(pathname)`**：对 `/chat`、`/knowledge`、`/tickets`、`/profile` 及其子路径（如 `/tickets/123`）返回 `true`。  
  - 登录、注册页不需登录即可访问。

- **`ProtectedRoute`**（`src/frontend/src/components/ProtectedRoute.tsx`）  
  - 包装需登录的整块布局（主布局及其子路由）。  
  - 若 **未登录**（`!getToken()`）且当前路径为 **`isAuthRequiredPath(location.pathname)`**：  
    - 使用 **`Navigate`** 跳转到 **`/login?redirect=${encodeURIComponent(当前完整 URL)}`**。  
    - 当前完整 URL 使用 **`location.pathname + location.search`**（同源内使用 path+search 即可，避免跨域问题）。  
  - 若已登录或当前为公开页，则直接渲染 `children`。

### 2. 登录成功后回跳

- **登录页包装**（`routes/index.tsx` 中的 **`LoginPageWrapper`**）：  
  - 从 **`useLocation().search`** 中读取 **`redirect`** 参数。  
  - 登录成功回调中：若存在 **`redirect`**，则 **`navigate(decodeURIComponent(redirect), { replace: true })`**；否则 **`navigate(ROUTES.CHAT, { replace: true })`**。  
- 注册页同理（**`RegisterPageWrapper`**），注册成功后按 `redirect` 或默认进入 `/chat`。

### 3. 401 与 SPA 内跳转

- **`src/frontend/src/services/api/config.ts`**  
  - 提供 **`setNavigateToLogin(fn)`**，由路由层（如 `AppRoutes`）注入 **`navigate`**。  
  - **`handleUnauthorized()`** 中：若无旧版「弹登录框」回调且已设置 **`navigateToLogin`**，则使用 **`navigateToLogin(\`/login?redirect=${encodeURIComponent(pathname+search)}\`)`**，实现 401 时无刷新跳转登录并带 redirect。

### 4. 验收要点

- 未登录访问 `/tickets` 或 `/tickets/123` 时，自动跳转到 `/login?redirect=...`，且 redirect 为编码后的目标 path+search。
- 登录成功后跳回 redirect 对应页面；无 redirect 时进入 `/chat`。
- 接口返回 401 时，若已配置 `setNavigateToLogin`，则跳转登录页并带当前 URL 的 redirect。
