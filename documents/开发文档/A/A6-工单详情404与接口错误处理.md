# A6：工单详情 404 与接口错误处理

## 修改点

> 对 `/tickets/:id` 等依赖后端数据的路由：请求详情接口若返回 404，展示 404 页或「工单不存在」等文案，不静默跳转首页。

## 实现说明

### 1. API 层 404 识别

- **`src/frontend/src/services/api/config.ts`**  
  - 定义 **`ApiError`** 类，包含 `code: number` 与 `message: string`，用于业务错误（含 404）的区分。

- **`src/frontend/src/services/api/request.ts`**  
  - 若 **HTTP 状态为 404**：先尝试解析 body 中的 `message`，否则使用默认「资源不存在」，然后 **`throw new ApiError(404, message)`**。  
  - 若响应体为 **`code === 404`**（与后端约定一致）：同样 **`throw new ApiError(404, result.message || '资源不存在')`**。  
  - 这样调用方可通过 `err instanceof ApiError && err.code === 404` 判断资源不存在。

### 2. 工单详情路由 `TicketDetailRoute`

- 位置：**`src/frontend/src/pages/TicketDetailRoute.tsx`**  
- 根据 `useParams().id` 调用 **`getTicketDetail(id)`**。  
- **成功**：将接口返回的 `TicketDetail` 映射为页面所需的 `Ticket` 类型，渲染 **`TicketDetailPage`**。  
- **捕获到 `ApiError` 且 `code === 404`**（或其他导致「工单不存在」的错误）：渲染 **`<NotFoundPage message="工单不存在" />`**，不跳转首页。  
- 其他错误（如网络异常）可同样展示 404 页或统一错误提示，由产品/联调约定；当前实现中对非 404 错误也展示「工单不存在」以避免静默跳转。

### 3. 验收要点

- 访问 `/tickets/不存在的id` 或后端返回 404 时，页面展示「工单不存在」及「返回首页」「返回工单列表」入口，不静默跳转到首页。
- 正常存在的工单 ID 仍正常展示详情页。
