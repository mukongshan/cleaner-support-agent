# **Cleaner - Support - Agent（清洁智能助手）项目背景文档**

## 一、项目概述

Cleaner - Support - Agent 是一款基于大语言模型（LLM）的垂直领域售后支持系统，旨在解决清洁机器人在商业场景应用中知识与经验传递效率低下的问题，降低经销商售中售后人力成本，打破厂商与经销商之间的知识鸿沟，最终构建清洁机器人领域的智能专家能力体系。

## 二、产品特性与行业现状

### （一）产品特性

本产品所支持的清洁机器人在实际清洁过程中，需要人工手持操作并全程参与作业，机器人本体不具备完全自主完成清洁任务的能力，清洁质量、安全性与作业效率高度依赖操作人员的理解与执行。

因此，项目中 Agent 的职责定位为辅助人工理解、判断与规范操作流程，而非替代人工决策、控制设备或自动执行清洁任务，所有涉及设备安全、不可逆操作、参数调优或维修判断的问题，均不在 Agent 的自主处理范围内。

### （二）行业现状与问题

随着清洁机器人在商业场景（如零售商业、办公写字楼、公共空间等）的逐步普及，影响其规模化落地的核心瓶颈正在从“设备能力”转向“知识与经验的传递效率”。从一线使用与服务支持的实际情况来看，当前主要存在以下系统性问题：

1. **一线人员能力门槛高**：清洁工文化水平参差不齐、人员流动性高，传统线下培训成本高、效率低，且难以标准化复制，新手上手难度大。
2. **场景与配置关系复杂**：不同空间结构、地面材质、污染类型与清洁设备的功能选型、作业方式及耗材使用之间存在复杂关联，缺乏经验易导致效果不佳或风险上升。
3. **售前支持高度依赖人工**：经销商对产品卖点、版本差异和适用场景理解不一致，售前咨询与培训高度依赖人工专家，沟通成本高且信息不一致。
4. **售后支持效率低**：大量简单、重复的解释性问题占用了售后专家时间，缺乏低成本的自助理解与分流机制。

## 三、核心问题总结

清洁机器人规模化应用的关键瓶颈，已从“设备是否足够智能”，转变为“知识与经验能否以低成本、高一致性的方式被沉淀并触达一线用户”。

## 四、项目愿景与目标

### （一）项目愿景

1. 短期：终端用户可以通过智能助手快速了解机器使用方法，在遇到机器故障时能快速获取相关信息；通过一个对话式智能 Agent，替代清洁机器人在售前咨询与基础使用阶段中的标准人工讲解与培训支持。
2. 长期：构建清洁机器人领域的智能专家能力体系，实现产品知识与行业经验的规模化复制。

### （二）项目目标

1. **MVP 目标**：通过一个对话式智能 Agent，替代清洁机器人在售前咨询与基础使用阶段中的标准人工讲解与培训支持。
2. **长期目标**：构建清洁机器人领域的智能专家能力体系，实现产品知识与行业经验的规模化复制。

## 五、设计前提与适用范围

### （一）人工在环的清洁作业前提

清洁机器人的实际作业依赖人工手持操作，这决定了 Agent 主要起到辅助作用，而非替代人工进行决策和操作。

### （二）文档目标与适用阶段

本文档用于定义清洁机器人智能 Agent 的产品定位与设计原则、MVP 阶段的能力范围、边界与验证目标以及 Agent 在不同阶段与角色下的运行约束模型。但不作为设备控制系统、售后工程诊断系统、自动化运维或专家系统的设计说明。

# 六、项目功能设计

## 核心功能
- **用户与身份**：手机号登录/注册，用户资料管理，角色区分（用户、工程师、管理员）。
- **智能问答（AI 助手）**：面向用户的清洁机器人使用与故障咨询，支持多轮会话。
- **工单管理**：创建、受理、分派、处理、反馈、归档全流程；工单优先级与状态跟踪；支持工单对话（用户↔工程师）。
- **知识/媒体中心**：维护知识文档、视频、PDF 等，支持分类、检索与详情查看。
- **媒体上传**：用户或管理员上传图片/视频等媒体资源，供工单或知识库使用。

## 业务流程概览
1) **咨询阶段**：用户向 AI 助手提问，获取使用/故障指导，必要时生成工单。  
2) **工单阶段**：用户提交工单并上传附件 → 管理员/客服受理 → 分派工程师 → 处理与对话沟通 → 完成或取消 → 用户反馈 → 归档。  
3) **知识复用**: 常见问题沉淀为知识/媒体内容，供搜索与自助查看；工单中的媒体上传可沉淀到知识库（经审核）。

## 角色与权限（业务视角）
- **用户**：登录/注册、AI 咨询、创建工单、查看/更新本人工单、提交工单反馈、查看知识/媒体。
- **工程师**：查看分配工单、与用户对话、更新进度、完成工单。
- **管理员/客服**：查看所有工单、受理与分派、设置工程师与预计时间、归档工单、维护知识/媒体内容。

## 价值与目标
- **效率**：AI 减少人工咨询量，优先级与分派机制缩短响应时间。
- **体验**：对话式沟通、附件支持与进度可视化，提升用户满意度。
- **沉淀**：知识与媒体资产持续积累，反哺自助与工程师处理效率。
