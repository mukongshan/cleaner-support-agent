# 售后问答机器人原型设计 - 页面设计报告

## 项目概述

本项目是一个面向智能扫地机器人设备的售后支持系统原型，采用移动端优先设计，提供AI智能问答、知识库浏览、工单管理、设备控制等核心功能。

**技术栈：** React 18 + TypeScript + Vite + Tailwind CSS + shadcn/ui + Framer Motion

---

## 整体架构

系统采用单页应用（SPA）架构，通过底部导航栏在5个主要页面间切换：

- **首页** - 设备状态监控与控制中心
- **问答** - AI智能售后助手对话界面
- **知识库** - 产品文档与教程中心
- **工单** - 服务工单管理
- **我的** - 个人中心与设备档案

## 一、首页（Home）

### 1.1 页面功能概述

首页是用户进入应用后的主界面，集中展示设备实时状态、提供快速控制入口和核心服务场景入口。

### 1.2 核心功能模块

#### 1.2.1 顶部状态栏
- **设备名称显示**：显示当前连接的设备名称（如"客厅的扫地僧 X10"）
- **网络状态指示**：
  - 在线/离线状态显示
  - WiFi信号强度可视化（4格信号条）
  - 离线时显示灰色离线图标
- **设备状态标签**：动态显示设备当前状态
  - 状态类型：清扫中、回充中、待机、故障
  - 不同状态对应不同颜色（绿色/蓝色/灰色/红色）

#### 1.2.2 核心场景入口区
提供三个主要的售后服务入口：

1. **我想去问**（蓝色渐变按钮）
   - 功能：跳转到AI智能问答页面
   - 图标：MessageSquare
   - 副标题：AI智能问答

2. **我想去查**（紫色渐变按钮）
   - 功能：跳转到知识库中心页面
   - 图标：BookOpen
   - 副标题：知识库中心

3. **紧急上报**（红色渐变按钮）
   - 功能：快速创建报障工单
   - 图标：Camera（带脉冲动画效果）
   - 副标题：拍照报障
   - 特殊功能：点击后跳转到问答页面并自动输入"紧急报障"

#### 1.2.3 快捷搜索框
- 位置：场景入口区下方
- 功能：快速搜索问题
- 交互：聚焦后自动跳转到问答页面
- 占位符提示："描述您的问题，如：机器人不充电..."

#### 1.2.4 故障报警区（条件显示）
- **触发条件**：当设备检测到异常时显示
- **显示内容**：
  - 红色警告边框和背景
  - AlertCircle图标
  - 异常标题："检测到设备异常"
  - 具体错误信息（如"悬空传感器异常"）
- **操作按钮**："立即咨询 AI 医生"
  - 点击后跳转到问答页面，自动输入错误信息

#### 1.2.5 机器视觉展示区
- **机器人图片展示**：
  - 圆形图片容器，带阴影效果
  - 支持图片加载失败的降级处理
- **动态状态效果**：
  - **清扫中**：双光圈脉冲动画 + 旋转边框动画
  - **回充中**：单光圈呼吸动画 + 中心闪电图标（带脉冲效果）
  - **故障状态**：红色光圈脉冲动画

#### 1.2.6 关键参数指标

**电量卡片**：
- 环形进度条显示电池电量百分比
- 电量图标（Battery）
- 颜色指示：>20%绿色，≤20%红色
- 预计可清扫面积显示

**清扫数据卡片**：
- 今日清扫面积（平方米）
- 清扫耗时（分钟）
- 蓝色圆点标识

#### 1.2.7 快捷操作区（控制组件）
位于关键参数指标下方，提供4个快速控制按钮：

1. **开始**（绿色）
   - 图标：Play
   - 功能：设置机器人状态为"清扫中"

2. **暂停**（灰色）
   - 图标：Pause
   - 功能：设置机器人状态为"待机"

3. **回充**（蓝色）
   - 图标：Home
   - 功能：设置机器人状态为"回充中"

4. **寻找**（紫色）
   - 图标：Radio
   - 功能：切换故障报警显示状态（测试功能）

所有按钮支持点击反馈动画和悬停效果。

#### 1.2.8 水箱状态卡片
- **清水箱**：
  - 蓝色进度条
  - 百分比显示
- **污水箱**：
  - 颜色指示：>80%红色，≤80%橙色
  - 百分比显示
- 水滴图标标识

### 1.3 交互特性

- 所有按钮支持触摸反馈（haptic-feedback）
- 场景入口按钮使用Framer Motion实现点击缩放动画
- 故障报警区支持滑入动画
- 状态变化时相关视觉元素平滑过渡

## 二、问答页面（ChatPage）

### 2.1 页面功能概述

问答页面是AI智能售后助手的主要交互界面，支持文字、图片、语音输入，提供智能问答服务，并具备完整的对话历史管理功能。

### 2.2 核心功能模块

#### 2.2.1 顶部工具栏

**聊天视图工具栏**：
- 标题："AI 智能售后助手"
- **历史记录按钮**（History图标）
  - 功能：切换到历史记录列表视图
  - 点击前会自动保存当前对话
- **新对话按钮**（RefreshCw图标）
  - 功能：清空当前对话，开始新对话
  - 会先保存当前对话到历史记录

**历史记录视图工具栏**：
- 返回按钮（ArrowLeft图标）：返回聊天视图
- 标题："历史记录"
- 新对话按钮：开始新对话

#### 2.2.2 对话区域

**消息展示**：
- **用户消息**：
  - 右对齐
  - 蓝色背景，白色文字
  - 圆角设计（右下角小圆角）
  - 时间戳右对齐显示

- **AI消息**：
  - 左对齐
  - 白色背景，灰色文字
  - 圆角设计（左下角小圆角）
  - AI头像（蓝色圆形背景，白色"AI"文字）
  - 时间戳左对齐显示

- **图片消息**：
  - 支持在消息中显示上传的图片
  - 图片自动适应宽度，圆角显示

- **引用来源**：
  - 显示知识库来源信息
  - 格式："来源：用户手册 - 维护保养 P.23-25"
  - FileText图标标识

**AI思考状态显示**：
- 显示AI处理中的步骤：
  1. 正在识别故障码...
  2. 正在查询维修知识库...
  3. 正在生成解决方案...
- 加载动画：Loader图标旋转
- 打字动画：三个圆点依次闪烁

**空状态**：
- 当没有对话记录时显示
- AlertCircle图标
- 提示文字："还没有对话记录"
- 引导文字："试试下面的常见问题，或者直接向我提问"

#### 2.2.3 猜你想问（快捷问题）

- **显示条件**：消息数量≤1且AI不在思考时
- **功能**：提供常见问题快捷入口
- **问题列表**：
  - 如何清理主刷？
  - 更换边刷教程
  - 机器人不回充怎么办？
  - 如何清洁传感器？
- **交互**：点击问题自动发送消息

#### 2.2.4 问题解决提示区

- **显示条件**：消息数量>3且AI不在思考且未创建工单
- **功能**：在多轮对话后引导用户
- **内容**：
  - 提示："问题解决了吗？"
  - 说明文字：如果方案未解决，可以创建工单
- **操作按钮**：
  - **已解决**（绿色按钮）：标记问题已解决，显示感谢消息
  - **创建工单**（蓝色按钮）：打开创建工单弹窗

#### 2.2.5 创建工单弹窗

- **触发方式**：点击"创建工单"按钮
- **弹窗内容**：
  - 标题："创建服务工单"
  - 关闭按钮（X图标）
  - **信息提示卡片**（蓝色背景）：
    - 工单将包含：问题描述和对话记录、上传的图片和视频、设备信息
  - **优先级选择**：
    - 三个选项：低、中、高
    - 默认选中"中"
    - 颜色区分：低（灰色）、中（蓝色）、高（红色）
  - **补充说明输入框**（可选）：
    - 多行文本输入
    - 占位符："补充其他信息..."
  - **确认创建按钮**：
    - 点击后创建工单
    - 显示工单编号（格式：WO + 时间戳后9位）
    - 提示："工程师将在2小时内与您联系"
    - 自动跳转到工单页面

#### 2.2.6 输入区域

**输入组件**：
- **图片上传按钮**（ImageIcon）：
  - 点击打开文件选择器
  - 支持图片格式
  - 上传后自动发送消息
  - 模拟图片分析功能

- **文字输入框**：
  - 占位符："描述您遇到的问题..."
  - 支持回车发送
  - 灰色背景，聚焦时显示蓝色边框

- **语音/发送按钮**：
  - 有文字时：显示发送按钮（Send图标，蓝色背景）
  - 无文字时：显示语音按钮（Mic图标，灰色背景）
  - **语音录制状态**：
    - 录音时显示红色背景和动画波浪线
    - 模拟语音识别功能

#### 2.2.7 历史记录列表视图

**列表项显示**：
- **对话卡片**：
  - 标题：对话的第一条用户消息（前20字符）
  - 消息数量显示
  - 更新时间显示（格式化：月日 时:分）
  - MessageSquare图标标识
- **删除按钮**：
  - 每个卡片右侧有删除按钮（Trash2图标，红色）
  - 点击删除对应历史记录
  - 如果删除的是当前对话，会重置对话

**空状态**：
- History图标
- 提示："暂无历史记录"
- 说明："开始对话后，历史记录会自动保存"
- "开始新对话"按钮

**交互功能**：
- 点击卡片：加载该历史记录到聊天视图
- 长按或点击删除按钮：删除历史记录
- 所有历史记录保存到localStorage，刷新后仍然保留

### 2.3 AI响应逻辑

系统根据用户输入的关键词提供智能回复：

- **主刷/清理**：返回主刷清理步骤（5步）
- **边刷**：返回边刷更换步骤（4步）
- **回充/充电**：返回回充问题排查步骤（4步）
- **传感器**：返回传感器清洁方法（3步）
- **其他问题**：引导用户提供更多信息（图片、错误代码、具体表现）

### 2.4 交互特性

- 消息发送后自动滚动到底部
- AI思考过程可视化
- 图片上传和分析功能
- 语音输入模拟
- 历史记录本地持久化存储
- 平滑的页面切换动画

## 三、知识库页面（KnowledgePage）

### 3.1 页面功能概述

知识库页面提供产品相关的文档、教程、视频等资料浏览和搜索功能，帮助用户自主解决问题。

### 3.2 核心功能模块

#### 3.2.1 顶部导航栏

- **返回按钮**（ArrowLeft图标）：返回首页
- **标题**："知识中心"
- **搜索框**：
  - 搜索图标（左侧）
  - 占位符："搜索教程、视频、文档..."
  - 实时搜索过滤
  - 灰色背景，聚焦时蓝色边框

#### 3.2.2 分类标签栏

- **分类类型**：
  1. **全部**（蓝色）：显示所有内容
  2. **操作指南**（蓝色）：产品使用说明
  3. **维护保养**（绿色）：设备维护教程
  4. **产品演示**（紫色）：功能演示视频

- **交互特性**：
  - 选中状态：带颜色的背景和文字
  - 未选中状态：灰色背景
  - 点击切换分类
  - 支持横向滚动（小屏幕）

#### 3.2.3 内容列表

**知识项卡片**：
- **图标区域**：
  - 渐变背景（蓝色系）
  - emoji图标（📱🧹🔧🔄等）
  - 圆角设计

- **内容区域**：
  - **标题**：知识项标题（加粗，截断过长内容）
  - **描述**：内容简介（最多2行，超出截断）
  - **元信息**：
    - 内容类型图标和文字（视频/PDF/图文）
    - 视频时长（Clock图标 + 时间，如"8:32"）
  - **右箭头图标**（ChevronRight）：指示可点击

- **交互特性**：
  - 悬停时阴影加深
  - 点击进入详情（当前为模拟）
  - 卡片进入动画（延迟依次显示）

**内容类型**：
- **图文**（Article）：Book图标
- **视频**（Video）：Video图标 + 时长显示
- **PDF**（PDF）：FileText图标

#### 3.2.4 空状态

- **触发条件**：搜索或筛选无结果
- **显示内容**：
  - Search图标（灰色）
  - 标题："未找到相关内容"
  - 提示："试试其他关键词或浏览不同分类"

#### 3.2.5 底部提示栏

- **内容**：
  - Download图标
  - 文字："所有资料支持离线下载和收藏"
- **功能说明**：提示用户资料可离线使用（当前为提示，未实现具体功能）

### 3.3 搜索与筛选

- **实时搜索**：输入关键词即时过滤
- **分类筛选**：点击分类标签切换内容
- **组合筛选**：搜索和分类可同时生效
- **搜索范围**：标题和描述

### 3.4 知识库内容示例

- 首次使用设置指南
- 日常清洁与维护
- 主刷拆卸与清理教程
- 边刷更换操作指南
- 全屋清扫功能演示
- 地图绘制与编辑
- 定时清扫设置教程
- 产品使用手册 PDF
- 故障诊断与排查
- 传感器清洁保养

### 3.5 交互特性

- 平滑的分类切换动画
- 卡片列表的延迟进入动画
- 响应式布局适配
- 搜索框实时过滤

## 四、工单页面（TicketsPage）

### 4.1 页面功能概述

工单页面用于管理用户提交的服务工单，支持查看工单状态、搜索筛选、创建新工单等功能。

### 4.2 核心功能模块

#### 4.2.1 顶部操作栏

- **标题**："我的工单"
- **创建工单按钮**：
  - 蓝色背景，白色文字
  - Plus图标 + "创建工单"文字
  - 点击后跳转到问答页面，自动输入"创建工单"

#### 4.2.2 搜索框

- 搜索图标（左侧）
- 占位符："搜索工单编号或问题描述..."
- 实时搜索过滤
- 搜索范围：工单编号、标题、描述

#### 4.2.3 统计卡片

三个统计卡片横向排列：

1. **全部工单**（蓝色渐变）
   - 显示总工单数量
   - 蓝色文字和背景

2. **处理中**（黄色渐变）
   - 显示处理中的工单数量
   - 黄色文字和背景

3. **已完成**（绿色渐变）
   - 显示已完成的工单数量
   - 绿色文字和背景

#### 4.2.4 状态过滤标签

- **全部**：显示所有状态的工单（显示总数）
- **待处理**（黄色）：仅显示待处理工单
- **处理中**（蓝色）：仅显示处理中工单
- **已完成**（绿色）：仅显示已完成工单
- **已取消**（灰色）：仅显示已取消工单（当前未在标签中显示）

- **交互特性**：
  - 选中状态：对应颜色的背景和白色文字
  - 未选中状态：灰色背景和文字
  - 每个标签显示对应状态的工单数量

#### 4.2.5 工单列表

**工单卡片**：
- **左侧边框**：根据状态显示不同颜色
  - 待处理：黄色
  - 处理中：蓝色
  - 已完成：绿色
  - 已取消：灰色

- **头部信息**：
  - **标题**：工单问题标题（加粗）
  - **图片标识**：如果有图片，显示Camera图标
  - **工单编号**：字体等宽显示（如"WO20240112001"）
  - **工单类型**：故障报修 / 问题咨询
  - **右箭头**：指示可点击查看详情

- **描述信息**：
  - 问题描述文字
  - 最多显示2行，超出截断

- **状态和优先级标签**：
  - **状态标签**：
    - 带图标的状态标识
    - 不同状态对应不同颜色和图标
    - 待处理：Clock图标，黄色
    - 处理中：AlertCircle图标，蓝色
    - 已完成：CheckCircle图标，绿色
    - 已取消：XCircle图标，灰色
  - **优先级标签**：
    - 高：红色
    - 中：橙色
    - 低：灰色

- **底部信息栏**：
  - **左侧**：
    - 创建日期（Calendar图标）
    - 分配工程师（UserIcon图标，如果有）
  - **右侧**：
    - 处理中工单：预计处理时间（Clock图标，蓝色）
    - 已完成工单：完成日期（CheckCircle图标，绿色）

- **快捷操作按钮**（仅处理中工单显示）：
  - **发送消息**（MessageSquare图标，蓝色背景）
  - **致电工程师**（Phone图标，绿色背景）

#### 4.2.6 空状态

- **触发条件**：筛选或搜索无结果
- **显示内容**：
  - FileText图标（灰色）
  - 标题："暂无工单记录"
  - 提示文字：
    - 有搜索条件时："未找到匹配的工单"
    - 无搜索条件时："遇到问题时，AI助手会为您创建工单"

## 七、设计规范

### 7.1 颜色系统

- **主色调**：蓝色系（#3b82f6）
- **成功/正常**：绿色系（#10b981）
- **警告**：黄色/橙色系（#f59e0b）
- **错误/危险**：红色系（#ef4444）
- **中性色**：灰色系（#6b7280, #9ca3af等）

### 7.2 字体规范

- **标题**：font-semibold（600）
- **正文**：font-medium（500）或默认（400）
- **小字**：text-xs（12px）或text-sm（14px）

### 7.3 间距规范

- **页面边距**：px-4（16px）
- **卡片内边距**：p-4（16px）
- **元素间距**：gap-2（8px）、gap-3（12px）、gap-4（16px）

### 7.4 圆角规范

- **卡片**：rounded-xl（12px）或rounded-2xl（16px）
- **按钮**：rounded-lg（8px）或rounded-xl（12px）
- **输入框**：rounded-xl（12px）

### 7.5 阴影规范

- **卡片阴影**：shadow-sm（轻微）或shadow-md（中等）
- **按钮悬停**：shadow-lg 到 shadow-xl

### 7.6 动画规范

- **页面切换**：使用Framer Motion实现平滑过渡
- **按钮点击**：scale动画（0.95缩放）
- **列表项**：延迟进入动画（stagger）
- **进度条**：从0到目标值的过渡动画

## 八、响应式设计

### 8.1 布局约束

- **最大宽度**：max-w-md（28rem，约448px）
- **居中显示**：mx-auto
- **移动端优先**：所有设计以移动端为基准

### 8.2 滚动处理

- **页面容器**：flex-1 overflow-y-auto
- **列表区域**：独立的滚动容器
- **安全区域**：支持iOS安全区域适配

### 8.3 触摸优化

- **按钮尺寸**：最小44x44px（触摸目标）
- **点击反馈**：haptic-feedback类名（模拟触觉反馈）
- **滑动操作**：支持横向滚动的标签栏
